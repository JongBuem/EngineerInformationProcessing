C 구조체 포인터 · 문자열 연산 추적

C · Struct · Pointer Arithmetic · String · Tracing
Q문제 코드
C 
#include <stdio.h>

struct Test { int i; const char *g; };

int main() {
    struct Test test[] = {
        {1, "AB"},
        {2, "DC"},
        {3, "EB"}
    };
    struct Test *p = &test[1];        // p → test[1]
    printf("%s", p->g + (p->i - 1));  // 핵심 연산
    return 0;
}

위 코드의 출력값은 무엇인가?

C
p→test[1] │ p->g = "DC" │ p->i - 1 = 1 │ "DC"+1 = "C"
① 문제 유형

C 구조체 배열 + 포인터 연산 + 문자열 포인터 산술을 한 번에 추적하는 문제입니다.
세 가지 개념이 하나의 식에 압축되어 있습니다.


  • 🔷구조체 배열 포인터 — *p가 배열 원소 중 어느 것을 가리키는지 파악합니다.
  • 🔷화살표 연산자 -> — 포인터로 구조체 멤버에 접근합니다.
  • 🔷문자열 포인터 산술 — const char *g는 문자열 첫 번째 문자를 가리키는 포인터이므로, 정수를 더하면 해당 오프셋 위치의 문자부터 출력됩니다.
② 배열 & 포인터 메모리 구조
③ 단계별 계산 과정
단계 식 값 설명
① p = &test[1] test[1] 주소 p는 {i=2, g="DC"}를 가리킴
② p->i 2 test[1].i = 2
③ p->i - 1 1 오프셋 계산 : 2 - 1 = 1
④ p->g "DC" (첫 문자 'D' 주소) test[1].g 는 문자열 "DC"의 시작 포인터
⑤ p->g + 1 'C' 주소 포인터를 1 증가 → 'C'를 가리킴
출력 printf("%s", ...) C %s 는 해당 주소부터 '\0'까지 출력 → "C\0" = C
④ 핵심 주의 포인트
  • 📌 const char *g는 문자열 리터럴을 가리키는 포인터입니다. 문자열 자체가 배열처럼 메모리에 존재하므로 정수를 더해 임의 위치부터 읽을 수 있습니다.
  • ➕ 포인터 + 정수 연산은 해당 타입 크기만큼 주소를 이동시킵니다. char*는 1바이트 단위이므로 +1은 정확히 다음 문자로 이동합니다.
  • 🖨️ %s 형식 지정자는 전달된 주소부터 '\0'을 만날 때까지 문자를 출력합니다. "DC"+1 → 'C', '\0' 순서이므로 "C"만 출력됩니다.
  • 🎯 p->i - 1 = 2 - 1 = 1이 오프셋입니다. 만약 i=1이었다면 +0이 되어 "DC" 전체가 출력됩니다.
