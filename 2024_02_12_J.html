Java 인터페이스 구현 + 홀수/짝수 합산 조건 추적

Java · Interface · implements · boolean flag · Modulo · Tracing
Q문제 코드
Java 
interface Number {
    int sum(int[] a, boolean odd);
}

public class Main {
    public static void main(String[] args) {
        int a[] = {1,2,3,4,5,6,7,8,9};
        OENumber OE = new OENumber();
        System.out.print(OE.sum(a, true) + ", " + OE.sum(a, false));
    }
}

class OENumber implements Number {
    @Override
    public int sum(int[] a, boolean odd) {
        int result = 0;
        for (int i = 0; i < a.length; i++) {
            // odd=true  → a[i] % 2 != 0 인 것만 (홀수)
            // odd=false → a[i] % 2 == 0 인 것만 (짝수)
            if ((odd && a[i] % 2 != 0) || (!odd && a[i] % 2 == 0))
                result += a[i];
        }
        return result;
    }
}

위 코드의 출력값은 무엇인가?

System.out.print() 출력 결과
25, 20
홀수 합 1+3+5+7+9=25  |  짝수 합 2+4+6+8=20
① 문제 유형

인터페이스를 구현한 클래스에서 boolean 플래그로 홀수/짝수 합산을 분기 처리하는 로직을 추적하는 문제입니다.


  • 🔷odd=true: a[i] % 2 != 0 — 홀수만 합산
  • 🔷odd=false: a[i] % 2 == 0 — 짝수만 합산
  • ⚠️조건식 주의: (odd && 조건A) || (!odd && 조건B) — odd가 true면 앞쪽, false면 뒤쪽만 활성화됩니다.

⑦ 핵심 주의 포인트
  • 🔌 인터페이스 Number를 OENumber가 구현(implements). 메서드 시그니처 int sum(int[] a, boolean odd)를 반드시 오버라이딩해야 합니다.
  • 🔀 boolean 플래그 분기: odd=true면 앞 조건, odd=false면 뒤 조건만 활성화됩니다. 단락 평가(short-circuit)로 불필요한 검사는 실행되지 않습니다.
  • 📊 홀수(1,3,5,7,9) 합 = 25, 짝수(2,4,6,8) 합 = 20
  • 🖨️ 최종 출력: "25, 20"
