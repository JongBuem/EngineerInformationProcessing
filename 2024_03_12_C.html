C 연결 리스트 인접 노드 값 교환 추적

C · Struct · Linked List · Pairwise Swap · Pointer · Tracing
Q문제 코드
C 
#include <stdio.h>

struct Node { int value; struct Node* next; };

void func(struct Node* node) {
    while (node != NULL && node->next != NULL) {
        int t = node->value;
        node->value       = node->next->value; // node와 node→next 값 교환
        node->next->value = t;
        node = node->next->next;               // 두 칸 이동
    }
}

int main() {
    struct Node n1 = {1, NULL};
    struct Node n2 = {2, NULL};
    struct Node n3 = {3, NULL};

    n1.next = &n3;   // ← 선언 순서와 다른 연결 순서! 함정
    n3.next = &n2;

    func(&n1);

    struct Node* current = &n1;
    while (current != NULL) {
        printf("%d", current->value);
        current = current->next;
    }
}

① 문제 유형

선언 순서와 연결(포인터) 순서가 다른 연결 리스트에서 인접 노드의 값을 교환하는 추적 문제입니다.


  • ⚠️핵심 함정: 노드는 n1, n2, n3으로 선언됐지만 연결 순서는 n1 → n3 → n2입니다. 선언 순서(1→2→3)로 착각하면 틀립니다.
  • 🔷func() 동작: 현재 노드와 다음 노드의 value를 교환한 뒤 node = node→next→next로 두 칸씩 이동합니다.
  • 🔷포인터(next)는 변경하지 않고 값(value)만 교환합니다.
② 초기 연결 리스트 구조 — 연결 순서 주의!

⑥ 핵심 주의 포인트
  • ⚠️ 연결 순서 ≠ 선언 순서: n1.next = &n3, n3.next = &n2이므로 리스트 순서는 n1 → n3 → n2입니다. n1→n2→n3이 아닙니다!
  • 🔄 값만 교환, 포인터 구조 불변: func()는 value만 바꾸고 next 포인터는 그대로 유지합니다.
  • ⏭️ node = node→next→next: 두 칸씩 이동합니다. 1회 후 n2에 도달하고 n2→next가 NULL이므로 루프 즉시 종료됩니다.
