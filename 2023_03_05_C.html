C 구조체 포인터 접근 연산자 빈칸 채우기

C · struct · pointer · dot(.) vs arrow(→) · malloc · Blank Fill
Q문제 코드
C 
#include <stdio.h>
#include <stdlib.h>

typedef struct Data {
    char c;
    int *numPtr;
} Data;

int main() {
    int num = 10;
    Data d1;
    Data *d2 = malloc(sizeof(Data));   // d2는 포인터!

    d1.numPtr = &num;
    d2->numPtr = &num;

    printf("%d\n", *d1.numPtr);
    printf("%d\n", *d2 빈칸 numPtr);   // ← 빈칸

    free(d2);
    return 0;
}
출력값:
10
10

빈칸에 들어갈 연산자는 무엇인가?

빈칸에 들어갈 연산자
→
d2는 포인터(Data *) → 멤버 접근 시 화살표 연산자(->) 사용
*d2->numPtr == *(d2->numPtr) == 10
① 문제 유형

구조체 변수(d1)와 구조체 포인터(d2)의 멤버 접근 방식 차이를 묻는 빈칸 채우기 문제입니다.


  • 📌d1은 구조체 변수 → 점 연산자 . 사용
  • 🎯d2는 구조체 포인터 (Data *d2) → 화살표 연산자 -> 사용

③ 점(.) vs 화살표(->) 연산자 비교
점 연산자 (.)
사용 조건: 구조체 변수
예시: d1.numPtr
의미: d1 자체가 구조체임
Data d1; → d1.멤버
화살표 연산자 (→)
사용 조건: 구조체 포인터
예시: d2->numPtr
의미: d2가 가리키는 구조체의 멤버
Data *d2; → d2->멤버
동치 관계:
d2->numPtr  ≡  (*d2).numPtr
*d2->numPtr  ≡  *(d2->numPtr)  = 10
④ printf 출력 단계 추적
printf 표현식 풀이 출력
1번째 *d1.numPtr d1.numPtr = &num → *(&num) = num = 10 10
2번째 *d2->numPtr d2->numPtr = &num → *(&num) = num = 10 10
⑤ 핵심 주의 포인트
  • 🎯 변수 선언만 보면 답이 보입니다: Data *d2 — 별표(*)가 있으면 포인터 → -> 사용
  • 🔗 ->는 (*d2).의 축약 표현입니다. 즉 포인터를 역참조한 뒤 멤버에 접근하는 두 동작을 하나로 합친 연산자입니다.
  • 📋 빈칸 정답: ->  |  완성된 코드: *d2->numPtr → 10 출력
