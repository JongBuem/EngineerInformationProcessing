Algorithm Study Note

C 연결 리스트 순회 & XOR 비트 연산 추적

C · Linked List · Pointer · Bitwise XOR · Tracing
Q문제 코드
C 
#include <stdio.h>

struct Node {
    struct Node* next;
    unsigned int x;
};

int main() {
    struct Node t1 = { 0, 5u  };   // next=NULL, x=5
    struct Node t2 = { 0, 7u  };   // next=NULL, x=7
    struct Node t3 = { 0, 11u };   // next=NULL, x=11

    t3.next = &t2;                  // t3 → t2
    t2.next = &t1;                  // t2 → t1 → NULL

    struct Node* curr = &t3;
    int sum = 0;

    while (curr) {
        sum = sum * 3 + curr->x;    // Horner 방식 누산
        curr = curr->next;
    }

    sum = (sum ^ 42u) + 100u;       // XOR 후 덧셈

    printf("%u\n", sum);
    return 0;
}

위 코드의 출력값은 무엇인가?

printf("%u\n", sum) 출력 결과
187
sum=125 → 125 XOR 42 = 87 → 87 + 100 = 187
① 문제 유형

연결 리스트를 포인터로 순회하며 누산한 뒤, XOR 비트 연산으로 최종값을 구하는 복합 추적 문제입니다.


  • 🔷연결 리스트 구조 파악 — next 포인터 연결 순서를 먼저 그려야 합니다. (t3 → t2 → t1 → NULL)
  • 🔷Horner 누산 공식 — sum = sum * 3 + x 패턴은 각 단계마다 이전 값을 3배 한 뒤 더합니다.
  • 🔷XOR(^) 비트 연산 — 같은 비트는 0, 다른 비트는 1이 됩니다. 이진수 변환 후 자리별로 계산해야 합니다.
② 연결 리스트 구조 & 순회 방향
③ while 루프 단계별 추적
반복 curr curr→x 계산식 sum 다음 curr
초기 — — sum = 0 0 &t3
1회 &t3 11 0 × 3 + 11 11 &t2
2회 &t2 7 11 × 3 + 7 = 33 + 7 40 &t1
3회 &t1 5 40 × 3 + 5 = 120 + 5 125 NULL
종료 NULL — while 조건 false 125 —
④ XOR 비트 연산 상세 계산

sum = (125 ^ 42) + 100

값 128 64 32 16 8 4 2 1 10진수
125 0 1 1 1 1 1 0 1 64+32+16+8+4+1
42 0 0 1 0 1 0 1 0 32+8+2
XOR (^) 같은 비트 → 0  |  다른 비트 → 1
= 87 0 1 0 1 0 1 1 1 64+16+4+2+1 = 87

최종:  87 + 100 = 187

⑤ 핵심 주의 포인트
  • 🔗 연결 리스트 방향 — t3.next = &t2, t2.next = &t1 이므로 순회 순서는 t3(11) → t2(7) → t1(5)입니다. 선언 순서와 다름에 주의하세요.
  • 🔢 Horner 누산 — sum = sum*3 + x는 매 단계마다 이전 합을 3배 합니다. 수식으로 표현하면 ((0×3+11)×3+7)×3+5 = 125입니다.
  • ⊕ XOR(^) 연산 — 비트 단위로 같으면 0, 다르면 1. 125(01111101) ^ 42(00101010) = 87(01010111).
  • 📝 원본 코드의 printf("%un", sum)은 %u\n의 역슬래시가 누락된 것으로, %u(unsigned int 출력) 형식입니다.
