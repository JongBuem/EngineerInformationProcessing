상속·오버라이딩 출력값 추적

Java · OOP · Inheritance · Method Overriding · Tracing
Q문제 코드
Java 
public class Main {
    public static void main(String[] args) {
        new Child();
        System.out.println(Parent.total);
    }
}

class Parent {
    static int total = 0;
    int v = 1;

    public Parent() {
        total += (++v);          // ① v 선증가 후 total에 더함
        show();                  // ② 오버라이딩된 Child.show() 호출!
    }

    public void show() {
        total += total;
    }
}

class Child extends Parent {
    int v = 10;               // Parent.v 와 별개의 필드

    public Child() {
        // super() 묵시적 호출 → Parent() 실행
        v += 2;               // ③ Child.v = 10+2 = 12
        total += v++;           // ④ total += 12, 이후 v = 13
        show();                 // ⑤ Child.show() 호출
    }

    @Override
    public void show() {
        total += total * 2;   // total = total + total*2 = total*3
    }
}

위 코드의 출력값은 무엇인가?

System.out.println(Parent.total) 출력 결과
54
Parent() : total=2→6 │ Child() : total=6+12=18→54
① 문제 유형

Java 상속·메서드 오버라이딩 상황에서 실행 순서와 출력값을 추적하는 문제입니다.
핵심 개념 세 가지가 복합적으로 출제됩니다.


  • 🔷묵시적 super() 호출 — Child 생성자 첫 줄에 자동으로 super()가 삽입됩니다.
  • 🔷동적 디스패치(Dynamic Dispatch) — Parent 생성자에서 show()를 호출해도, 실제 객체 타입이 Child이므로 Child.show()가 실행됩니다.
  • 🔷필드 섀도잉(Field Shadowing) — Child의 int v = 10은 Parent의 int v = 1과 완전히 별개입니다. ++v는 Parent.v에만 영향을 줍니다.
② 클래스 구조 & 실행 흐름
③ 단계별 변수 추적
단계 실행 구문 설명 Parent.v Child.v total
초기 new Child() 객체 생성 시작 1 10 0
① super() 묵시 호출 Child 생성자 → Parent() 먼저 실행 1 10 0
② total += (++v) Parent.v 선증가(1→2), total에 2 더함 2 10 2
③ show() → Child.show() 동적 디스패치 — Child의 show() 실행!
total += total * 2 → 2 + 2*2 = 6 		2 10 6
④ Parent() 종료 Child() 본체로 복귀 2 10 6
⑤ v += 2 Child.v = 10 + 2 = 12 2 12 6
⑥ total += v++ total에 현재 v(12)를 더한 뒤 v 후증가(12→13)
total = 6 + 12 = 18 		2 13 18
⑦ show() → Child.show() total += total * 2 → 18 + 36 = 54 2 13 54
출력 Parent.total static 필드, 최종값 출력 — — 54
④ 핵심 주의 포인트
  • ⚡ Parent() 내 show() 호출은 Child.show() — 생성자 내부라도 동적 디스패치가 적용됩니다. 실제 객체가 Child이므로 오버라이딩된 메서드가 호출됩니다.
  • 🗂️ v 필드는 두 개 — ++v는 Parent.v(=1→2)에만 적용됩니다. Child.v(=10)는 영향을 받지 않습니다.
  • ➕ Child.show() 공식 — total += total * 2는 곧 total = total * 3과 동일합니다. 2번 호출되므로 각각 ×3이 적용됩니다.
  • 🔢 v++ (후위 증가) — total += v++에서 total에는 증가 전 값(12)이 더해지고, 이후 v가 13이 됩니다.
