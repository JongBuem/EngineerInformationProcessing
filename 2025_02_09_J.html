Java 함수형 인터페이스 · 람다 · 예외 처리 추적

Java · Functional Interface · Lambda · try-catch · Exception · Tracing
Q문제 코드
Java 
public class Main {

    static interface F {
        int apply(int x) throws Exception;
    }

    public static int run(F f) {
        try {
            return f.apply(3);      // 항상 x=3 으로 호출
        } catch (Exception e) {
            return 7;              // 예외 발생 시 7 반환
        }
    }

    public static void main(String[] args) {
        F f = (x) -> {
            if (x > 2) {
                throw new Exception();  // x=3 → 3>2 → throw!
            }
            return x * 2;
        };
        System.out.print(run(f) + run((int n) -> n + 9));
    }
}

위 코드의 출력값은 무엇인가?

System.out.print() 출력 결과
19
run(f)=7 (예외→catch) + run(n→n+9)=12 → 7+12=19
① 문제 유형

함수형 인터페이스(람다)와 try-catch 예외 처리가 결합된 실행 흐름 추적 문제입니다.


  • 🔷함수형 인터페이스 — 추상 메서드가 하나뿐인 인터페이스. 람다로 구현 가능합니다.
  • 🔷run(F f)은 항상 x=3으로 호출하며, 예외 발생 시 7을 반환합니다.
  • 🔷두 번의 run() 반환값을 더해서 출력합니다. 각각 독립적으로 추적해야 합니다.
② 두 번의 run() 호출 흐름
③ 단계별 실행 추적
호출 x/n 값 람다 내부 동작 예외 반환값
run(f) x = 3 3 > 2 → true → throw Exception() 발생 O catch → 7
run(n→n+9) n = 3 3 + 9 = 12 없음 12
7 + 12 = 19
④ 함수형 인터페이스 & 람다 관계
구성 요소 역할 이 코드에서
interface F 함수형 인터페이스 선언 apply(int x) throws Exception
F f = (x) -> {...} 람다로 F 구현 x > 2이면 예외, 아니면 x*2 반환
(int n) -> n+9 익명 람다로 F 구현 n을 받아 n+9 반환 (예외 없음)
run(F f) F를 받아 apply(3) 실행 예외 → 7 반환 / 정상 → 결과값 반환
⑤ 핵심 주의 포인트
  • 📌 run()은 항상 x=3으로 람다를 호출합니다. 람다 내 매개변수명(x, n)과 무관하게 값은 3입니다.
  • 💥 첫 번째 람다: 3 > 2는 참이므로 Exception이 throw됩니다. catch 블록에서 7 반환.
  • ✅ 두 번째 람다: n + 9 = 3 + 9 = 12. 예외 없이 정상 반환. 12 반환.
  • ➕ run(f) + run(n→n+9) = 7 + 12 = 19
  • ⚠️ 람다가 throws Exception을 선언해도 반드시 던지는 게 아닙니다. 조건에 따라 예외 발생 여부가 결정됩니다.
