Java String.equals() vs == 참조 비교 추적

Java · String Pool · equals() · new String() · Reference · Tracing
Q문제 코드
Java 
public class Main {
    static void func(String[] m, int n) {
        /* ① 인접 원소 비교 루프 */
        for (int i = 1; i < n; i++) {
            if (m[i-1].equals(m[i])) {    // 내용(값) 비교
                System.out.print("O");
            } else {
                System.out.print("N");
            }
        }
        /* ② 원소 출력 루프 */
        for (String mo : m) {
            System.out.print(mo);
        }
    }

    public static void main(String[] args) {
        String[] m = new String[3];
        m[0] = "A";               // String Pool 참조
        m[1] = "A";               // 동일 Pool 객체 참조
        m[2] = new String("A"); // Heap에 새 객체 생성!
        func(m, 3);
    }
}

위 코드의 출력값은 무엇인가?

System.out.print 출력 결과
OOAAA
equals()는 내용 비교 → 모두 "A" → OO 출력 후 AAA 출력
① 문제 유형

String 리터럴(Pool)과 new String()(Heap)의 참조 차이를 이해하고, equals()가 내용 비교임을 파악하는 문제입니다.


  • ⚠️핵심 함정: new String("A")는 힙에 별도 객체를 생성하므로 ==로 비교하면 false가 됩니다. 그러나 이 코드는 ==가 아닌 .equals()를 사용합니다.
  • ✅.equals()는 내용(문자열 값)을 비교하므로 참조가 달라도 내용이 같으면 true를 반환합니다.
② 메모리 구조 — String Pool vs Heap
③ equals() vs == 차이 비교
✅ .equals() — 이 코드에서 사용
내용(문자열 값) 비교
"A".equals("A") → true
m[0].equals(m[1]) → true
m[1].equals(m[2]) → true
❌ == — 이 코드에서 미사용 (함정)
참조(주소) 비교
m[0] == m[1] → true (같은 Pool)
m[1] == m[2] → false (다른 객체)
new String()이 Heap에 새 생성!
④ 루프 ① — 인접 비교 (i=1, 2)
i m[i-1] m[i] 비교식 결과 출력
1 m[0] = "A" m[1] = "A" "A".equals("A") true O
2 m[1] = "A" m[2] = "A" (new) "A".equals("A") true O

→ 출력: OO

⑤ 루프 ② — 배열 원소 출력 (enhanced for)
순서 mo 출력
1번째 m[0] = "A" A
2번째 m[1] = "A" A
3번째 m[2] = "A" A

→ 출력: AAA

⑥ 전체 출력 순서 요약
⑦ 핵심 주의 포인트
  • 🏊 String Pool: "A" 리터럴은 JVM이 풀에 하나만 만들고 재사용합니다. m[0]과 m[1]은 동일한 객체를 가리킵니다.
  • 🆕 new String("A")는 힙에 별도 객체를 강제 생성합니다. ==로 비교하면 false이지만, .equals()는 내용을 비교하므로 true입니다.
  • ✅ .equals()를 사용했으므로 m[1]과 m[2]의 비교도 true → 모두 "O"를 출력합니다.
  • 🖨️ 최종 출력: "OOAAA" (줄바꿈 없음, print)
