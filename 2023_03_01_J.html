Java super.draw() 내부 동적 디스패치 추적

Java · super.draw() · Dynamic Dispatch · Overriding · Method Call Chain · Tracing
Q문제 코드
Java 
public class main {
    public static void main(String[] args) {
        A b = new B();   // 참조 타입: A, 실제 객체: B
        b.paint();
        b.draw();
    }
}

class A {
    public void paint() {
        System.out.print("A");
        draw();           // this.draw() → 동적 디스패치
    }
    public void draw() {
        System.out.print("B");
    }
}

class B extends A {
    public void paint() {
        super.draw();    // ★ A.draw() 강제 직접 호출
        System.out.print("C");
        this.draw();     // B.draw() 호출
    }
    public void draw() {
        System.out.print("D");
    }
}

위 코드의 출력값은 무엇인가?

출력 결과
BCDD
b.paint() → B(A.draw) C D  |  b.draw() → D
① 문제 유형 & 핵심 포인트

super.draw()로 부모의 A.draw()를 직접 호출하며, A.draw()는 단순히 print("B")만 실행하고 종료됨. 내부 draw() 재호출 없음입니다.

→ B.paint() 복귀 후 print("C"), this.draw() 순서로 진행

⑥ super.paint() vs super.draw() 핵심 비교
super.paint() 였다면
A.paint() → print("A")
→ draw() → B.draw() → "D"
→ print("C") → "C"
→ this.draw() → "D"
→ b.draw() → "D"
→ ADCDD
super.draw() (현재 코드)
A.draw() → print("B")
→ (A.draw엔 내부 draw 없음)
→ print("C") → "C"
→ this.draw() → "D"
→ b.draw() → "D"
→ BCDD
⑦ 핵심 주의 포인트
  • 🎯 super.draw()는 A.draw()를 강제 직접 호출합니다. A.draw()의 코드는 print("B") 하나뿐이므로 B만 출력하고 종료됩니다.
  • ⚠️ 만약 A.draw() 내부에 draw() 재호출이 있었다면 그것도 B.draw()로 동적 디스패치되었을 것입니다. (this=B이므로)
  • 📋 b.paint() = BCD  |  b.draw() = D  |  전체 = BCDD
