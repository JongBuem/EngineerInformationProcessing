Java 다중 catch + finally 예외 처리 흐름 추적

Java · Exception · try-catch · NullPointerException · finally · Tracing
Q문제 코드
Java 
public class Main {
    public static void main(String[] args) {
        int sum = 0;
        try {
            func();                          // NullPointerException 발생!
        } catch(NullPointerException e) { // ① 첫 번째 catch — 일치!
            sum = sum + 1;
        } catch(Exception e) {          // ② 두 번째 catch — 건너뜀
            sum = sum + 10;
        } finally {                       // ③ finally — 항상 실행!
            sum = sum + 100;
        }
        System.out.print(sum);
    }

    static void func() throws Exception {
        throw new NullPointerException();  // 예외 던짐
    }
}

위 코드의 출력값은 무엇인가?

System.out.print(sum) 출력 결과
101
0 + 1(NullPointerException catch) + 100(finally) = 101
① 문제 유형

다중 catch 블록에서 예외 타입 매칭 우선순위와 finally의 무조건 실행을 이해하는 예외 처리 흐름 추적 문제입니다.


  • ⚠️catch는 위에서 아래로 순서대로 매칭합니다. 첫 번째 일치하는 catch만 실행되고 나머지는 건너뜁니다.
  • 🔵finally는 예외 발생 여부와 관계없이 항상 마지막에 실행됩니다.
  • 🔷NullPointerException은 Exception의 하위 클래스이지만, 더 구체적인 catch가 먼저 있으면 그쪽이 처리합니다.
② Java 예외 클래스 계층 구조
Throwable
└── Exception
└── RuntimeException
└── NullPointerException ← 이번에 발생한 예외

NullPointerException ⊂ RuntimeException ⊂ Exception — 모두 상위 타입이지만 catch 우선순위는 위에서 아래

③ 실행 흐름 단계별 추적
1
try 블록 진입
sum = 0 → func() 호출
2
NullPointerException 발생!
throw new NullPointerException()
try 블록 즉시 중단 → catch 탐색 시작
3
catch(NullPointerException e) — ✅ 일치!
NullPointerException == NullPointerException → 매칭
sum = 0 + 1 = 1
4
catch(Exception e) — ⏭ 건너뜀
이미 위 catch에서 처리됨 → 실행 안 됨
sum 변화 없음 (여전히 1)
5
finally — 항상 실행!
예외 처리 여부와 무관하게 반드시 실행
sum = 1 + 100 = 101
6
System.out.print(sum)
101 출력
④ sum 변화 단계별 추적표
단계 블록 조건 연산 sum
초기화 main() — int sum = 0 0
catch ① NullPointerException ✅ 타입 일치 sum = 0 + 1 1
catch ② Exception ⏭ 건너뜀 (실행 안 됨) 1 (불변)
finally finally 무조건 실행 sum = 1 + 100 101
출력 print — System.out.print(sum) 101
sum = 0 + 1 (NullPointerException catch) + 100 (finally) = 101
⑤ 다중 catch 우선순위 규칙
  • 📌 catch 블록은 위에서 아래로 순서대로 타입을 비교합니다. 첫 번째 일치 시 실행 후 나머지 catch는 모두 건너뜁니다.
  • ⚠️ 만약 순서가 반대였다면 (catch(Exception e)가 먼저) → Exception이 NullPointerException을 포함하므로 +10이 되고 sum = 110이 됩니다.
  • 🔵 finally는 예외 발생 여부, catch 성공 여부와 무관하게 항상 실행됩니다. return 문이 있어도 finally는 실행됩니다.
