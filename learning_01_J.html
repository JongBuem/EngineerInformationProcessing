1클래스 구조 — 필드 · 생성자 · 메서드

Java 클래스는 필드(Field), 생성자(Constructor), 메서드(Method)로 구성됩니다. 각 역할을 명확히 구분하는 것이 OOP의 출발점입니다.

Java — 클래스 기본 구조 
class Person {
    // ① 필드 (멤버 변수)
    String name;
    int    age;

    // ② 생성자 — 클래스 이름과 동일, 반환 타입 없음
    Person(String name, int age) {
        this.name = name;   // this = 현재 인스턴스
        this.age  = age;
    }

    // ③ 메서드
    void greet() {
        System.out.println("Hi, I'm " + name);
    }
}

// 사용
Person p = new Person("Alice", 20);
p.greet();  // Hi, I'm Alice
구성 요소 특징 this 활용
필드 (Field) 클래스 내 변수. 인스턴스마다 별도 저장 this.name — 같은 이름 매개변수와 구분
생성자 (Constructor) 클래스 이름 = 메서드명, 반환 타입 없음 this() — 다른 생성자 호출 (체이닝)
메서드 (Method) 기능 정의, 반환 타입 명시 필수 this.method() — 현재 인스턴스 메서드 호출
static 멤버 인스턴스 없이 클래스로 직접 접근 static 내부에서 this 사용 불가 ❌
2접근 제어자 — public · private · protected · default

접근 제어자는 어디서 해당 멤버에 접근할 수 있는지를 결정합니다. 시험에서는 상속 관계에서 어느 제어자가 자식에게 보이는지가 자주 출제됩니다.

제어자 같은 클래스 같은 패키지 자식 클래스 외부
public ✅ ✅ ✅ ✅
protected ✅ ✅ ✅ ❌
default (없음) ✅ ✅ ❌ ❌
private ✅ ❌ ❌ ❌
Java 
class Animal {
    public    String name;     // 어디서나 접근 가능
    protected int    age;      // 자식까지 접근 가능
    private   double weight;   // 이 클래스 안에서만
    String type;               // default — 같은 패키지만
}

class Dog extends Animal {
    void show() {
        name   = "Rex";   // ✅ public
        age    = 3;       // ✅ protected
        // weight = 5.0;  ❌ private — 접근 불가
    }
}
3상속 — extends & super

상속(Inheritance)은 부모 클래스의 필드·메서드를 자식 클래스가 물려받는 것입니다. Java는 단일 상속만 지원하며(extends는 하나만), super로 부모에 접근합니다.

Asuper 키워드 사용법
Java 
class Animal {
    String sound = "...";
    Animal(String s) { sound = s; }
    void speak() { System.out.println(sound); }
}

class Dog extends Animal {
    Dog() {
        super("Woof");  // ① 부모 생성자 호출 — 반드시 첫 줄
    }
    void speak() {
        super.speak();    // ② 부모 메서드 호출
        System.out.println("I'm a dog");
    }
    void showSound() {
        System.out.println(super.sound); // ③ 부모 필드 접근
    }
}
B생성자 호출 규칙 ★ 시험 핵심
  • 🔴자식 생성자에 super()나 this()가 없으면 컴파일러가 super() 를 자동 삽입합니다.
  • 🔴super()와 this()는 반드시 생성자의 첫 줄에 와야 하며, 동시에 사용 불가합니다.
  • 🔴즉, new Child() 호출 시 항상 Parent 생성자가 먼저 완료된 후 Child 생성자 본문이 실행됩니다.
4오버라이딩 vs 오버로딩 ★ 시험 최빈출
구분 오버라이딩 (Overriding) 오버로딩 (Overloading)
정의 부모 메서드를 자식이 재정의 같은 이름 메서드를 매개변수 다르게 정의
조건 시그니처(이름+매개변수) 동일해야 함 매개변수 타입/개수가 달라야 함
반환 타입 동일 (또는 공변 반환 타입) 달라도 되지만 반환만 다르면 오버로딩 안 됨
어노테이션 @Override 권장 별도 어노테이션 없음
결정 시점 런타임 (동적 디스패치) 컴파일 타임 (정적 결정)
Java — 두 개념 비교 
class Base {
    void show()           { System.out.print("Base"); }
    void show(int n)       { System.out.print("Base+"+n); } // 오버로딩
}

class Child extends Base {
    @Override
    void show() { System.out.print("Child"); }  // 오버라이딩
}

Base obj = new Child();
obj.show();     // → "Child"  (런타임에 Child 버전 실행)
obj.show(5);   // → "Base+5" (Base 오버로딩 버전)

⚠ 함정: 참조 타입이 Base여도 오버라이딩된 메서드는 실제 객체(Child) 버전이 호출됩니다. 오버로딩된 메서드는 컴파일 타임 참조 타입 기준으로 결정됩니다.

5abstract 클래스 & interface
Aabstract 클래스
Java 
abstract class Shape {
    String color = "red";          // 일반 필드 OK
    abstract double area();        // 추상 메서드 — 구현 없음
    void printColor() {            // 일반 메서드 OK
        System.out.println(color);
    }
}

class Circle extends Shape {
    double r;
    Circle(double r) { this.r = r; }
    @Override
    double area() { return 3.14 * r * r; } // 반드시 구현
}
Binterface
Java 
interface Flyable {
    void fly();                     // 기본 public abstract
    default void land() {          // Java 8+ default 메서드
        System.out.println("landing");
    }
}

class Bird implements Flyable {  // ← implements 키워드
    public void fly() { System.out.println("flap!"); }
}
구분 abstract class interface
키워드 extends implements
다중 사용 ❌ 단일만 ✅ 여러 개 가능
필드 일반 필드 가능 상수만 (public static final)
생성자 가능 (직접 인스턴스화 불가) 없음
메서드 abstract + 일반 혼용 기본 abstract (default/static 추가 가능)
6시험 핵심 요약
  • ⭐this = 현재 인스턴스 참조 / this() = 같은 클래스의 다른 생성자 호출
  • ⭐super = 부모 참조 / super() = 부모 생성자 호출 — 자식 생성자 첫 줄
  • ⭐오버라이딩 → 런타임 결정 (실제 타입 기준) / 오버로딩 → 컴파일 결정 (참조 타입 기준)
  • ⭐abstract 클래스 → extends / interface → implements
  • ⭐static 메서드 안에서는 this, 인스턴스 필드 접근 모두 불가
