Algorithm Study Note

Java 값 전달 vs 참조 전달 추적

Java · Pass-by-Value · Array Reference · String · Tracing
Q문제 코드
Java 
public class Main {

    public static void change(String[] data, String s) {
        data[0] = s;       // 배열 내부 원소 변경 → 원본에 반영 O
        s = "Z";           // 지역 변수 s 재할당 → 원본에 반영 X
    }

    public static void main(String[] args) {
        String data[] = { "A" };
        String s = "B";
        change(data, s);
        System.out.print(data[0] + s);
    }
}

위 코드의 출력값은 무엇인가?

System.out.print() 출력 결과
BB
data[0]="B" (배열 원소 변경 반영) │ s="B" (재할당 미반영)
① 문제 유형

Java의 "값 전달(Pass-by-Value)" 원칙에서 배열과 기본 참조 변수의 동작 차이를 묻는 추적 문제입니다.


  • 🔷Java는 항상 값으로 전달합니다. 배열도 "배열 자체"가 아닌 배열의 주소(참조값)가 복사되어 전달됩니다.
  • 🔷data[0] = s — 복사된 주소를 통해 배열 내부 원소를 수정하므로 원본에 반영됩니다.
  • 🔷s = "Z" — 매개변수 s는 지역 복사본이므로 재할당해도 원본 main의 s에는 영향 없음
② 메모리 구조 — 호출 전 / change() 실행 중
③ 단계별 변수 추적
단계 실행 구문 main: data[0] main: s change: s
초기 data={"A"}, s="B" "A" "B" —
① change(data, s) 호출 "A" "B" "B" (복사)
② data[0] = s "B" ← 반영! "B" "B"
③ s = "Z" "B" "B" 그대로 (미반영) "Z" (지역만)
④ change() 종료 "B" "B" 소멸
출력 data[0] + s "B" "B" → "BB"
④ 배열 원소 수정 vs 참조 재할당 비교
구문 동작 원본 반영 이유
data[0] = s 배열 내부 원소 수정 반영 O 복사된 주소가 같은 배열을 가리키기 때문
s = "Z" 지역 변수 재할당 반영 X 지역 복사본만 다른 객체를 가리키게 됨
⑤ 핵심 주의 포인트
  • 📌 Java는 항상 값으로 전달(Pass-by-Value)합니다. 배열도 "주소값"이 복사되어 전달됩니다.
  • 🗂️ 복사된 주소로 배열 내부 원소를 수정하면 원본에 반영됩니다. 같은 힙 주소를 가리키기 때문입니다.
  • ✂️ 매개변수 s = "Z"는 로컬 참조 변수만 바꿉니다. main의 s는 여전히 "B"를 가리킵니다.
  • 🖨️ 결과: data[0] = "B", main의 s = "B" → 출력 "BB"
System.out.print(data[0] + s) 출력 결과
