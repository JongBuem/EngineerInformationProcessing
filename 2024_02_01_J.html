Java 배열 참조 동등성 비교 (== vs Arrays.equals)

Java · Array · Reference Equality · == · Heap · Tracing
Q문제 코드
Java 
public class Main {
    public static void check(int[] x, int[] y) {
        if (x == y) System.out.print("O");  // 참조(주소) 비교!
        else        System.out.print("N");
    }

    public static void main(String[] args) {
        int a[] = new int[] {1, 2, 3, 4};  // 힙에 새 배열 @A
        int b[] = new int[] {1, 2, 3, 4};  // 힙에 새 배열 @B (내용 같지만 다른 객체!)
        int c[] = new int[] {1, 2, 3};    // 힙에 새 배열 @C
        check(a, b);  // @A == @B ?
        check(b, c);  // @B == @C ?
        check(a, c);  // @A == @C ?
    }
}

① 문제 유형

Java 배열에서 == 연산자가 내용이 아닌 참조(메모리 주소)를 비교함을 이해하는 문제입니다.


  • ⚠️핵심 함정: a, b는 내용이 {1,2,3,4}로 동일하지만, new int[]로 각각 선언했으므로 힙에 서로 다른 객체가 생성됩니다.
  • 🔴배열에서 ==는 참조(주소) 비교입니다. 내용 비교가 필요하면 Arrays.equals()를 사용해야 합니다.
  • 🔷a, b, c는 모두 new int[]로 생성된 독립적인 객체이므로 세 비교 모두 false → 모두 "N".
② 힙(Heap) 메모리 구조
STACK
a→ @0x1A00
b→ @0x2B00
c→ @0x3C00
각 변수는 힙 주소를 저장
주소값이 모두 다름!

③ == vs Arrays.equals() 차이
⚠️ == 연산자 — 이 코드에서 사용
참조(주소) 비교
a == b → false (0x1A00 ≠ 0x2B00)
b == c → false (0x2B00 ≠ 0x3C00)
a == c → false (0x1A00 ≠ 0x3C00)
✅ Arrays.equals() — 내용 비교
원소 값 순서대로 비교
Arrays.equals(a,b) → true
Arrays.equals(b,c) → false (길이 다름)
Arrays.equals(a,c) → false (길이 다름)

⑥ 핵심 주의 포인트
  • 📦 Java에서 new int[]{ }는 항상 힙에 새로운 배열 객체를 생성합니다. 내용이 완전히 같아도 다른 주소를 가집니다.
  • 🔴 배열에서 ==는 참조(주소) 비교입니다. 세 배열이 모두 new int[]로 따로 생성됐으므로 어떤 쌍도 주소가 같지 않습니다.
  • ✅ 내용을 비교하려면 Arrays.equals(a, b)를 사용해야 합니다. 이 경우 a와 b는 true, b/c와 a/c는 false입니다.
