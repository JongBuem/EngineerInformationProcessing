C 연결 리스트 포인터 재연결 추적

C · Struct · Linked List · Pointer Reassignment · Tracing
Q문제 코드
C 
#include <stdio.h>
#include <stdlib.h>

struct node {
    int p;
    struct node* n;
};

int main() {
    struct node a = {1, NULL};
    struct node b = {2, NULL};
    struct node c = {3, NULL};

    /* ── 1차 연결 (이후 덮어씀) ── */
    a.n = &b;  b.n = &c;  c.n = NULL;

    /* ── 2차 연결 (최종 상태) ── */
    c.n = &a;  a.n = &b;  b.n = NULL;

    struct node* head = &c;
    printf("%d %d %d",
        head->p,
        head->n->p,
        head->n->n->p);
    return 0;
}

위 코드의 출력값은 무엇인가?

printf 출력 결과
3 1 2
head→c(3) → a(1) → b(2) → NULL
① 문제 유형

연결 리스트의 포인터를 두 차례 재연결한 뒤 최종 상태를 추적하는 문제입니다.


  • 🔷덮어쓰기 주의 — 1차 연결 후 2차 연결로 일부 포인터가 완전히 덮어써집니다. 최종 상태만이 유효합니다.
  • 🔷head = &c — 탐색은 c노드부터 시작합니다.
  • 🔷head->n->n->p — 화살표 연산자(->)를 3번 체인으로 따라가야 합니다.
② 1차 연결 → 2차 연결(최종) 변화
③ 포인터 최종 상태 (2차 연결 후)
노드 p (데이터) 1차 n 2차 n (최종) 변경 여부
a 1 &b &b 동일 (변경 없음)
b 2 &c NULL 변경됨! &c → NULL
c 3 NULL &a 변경됨! NULL → &a
④ printf 체인 역참조 단계별 추적
식 분해 현재 노드 값
head &c c —
head->p c.p c 3
head->n c.n = &a a —
head->n->p a.p a 1
head->n->n a.n = &b b —
head->n->n->p b.p b 2
⑤ 핵심 주의 포인트
  • ⚠️ 1차 연결 a→b→c→NULL은 이후 2차 연결로 b.n, c.n이 덮어써집니다. 최종 상태만 유효합니다.
  • 🔗 최종 연결: c → a → b → NULL. head=&c이므로 탐색은 c에서 시작합니다.
  • 🔢 head→p=3, head→n→p = a.p = 1, head→n→n→p = b.p = 2
  • 🖨️ 최종 출력: 3 1 2
