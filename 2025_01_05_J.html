Java 예외 처리 (try-catch-finally) 실행 흐름 추적

Java · Exception · try-catch-finally · ArithmeticException · Tracing
Q문제 코드
Java 
public class Main {
    public static void main(String[] args) {
        int a = 5, b = 0;

        try {
            System.out.print(a / b);        // 5 ÷ 0 → ArithmeticException 발생!
        } catch (ArithmeticException e) {
            System.out.print("출력1");      // ← 이 블록 실행
        } catch (ArrayIndexOutOfBoundsException e) {
            System.out.print("출력2");      // 건너뜀
        } catch (NumberFormatException e) {
            System.out.print("출력3");      // 건너뜀
        } catch (Exception e) {
            System.out.print("출력4");      // 건너뜀
        } finally {
            System.out.print("출력5");      // 항상 실행
        }
    }
}

① 문제 유형

try-catch-finally 예외 처리 구조에서 실행 흐름을 추적하는 문제입니다.


  • 🔴5 / 0 → 정수 나눗셈에서 0으로 나누면 ArithmeticException이 발생합니다.
  • 🟢catch 블록은 위에서부터 순서대로 타입을 비교합니다. 처음 일치한 블록 하나만 실행되고 나머지는 건너뜁니다.
  • 🟡finally 블록은 예외 발생 여부와 관계없이 항상 마지막에 실행됩니다.
② 예외 클래스 계층 구조 (상속 관계)
③ 실행 흐름 단계별 추적
①
a / b = 5 / 0 실행 → ArithmeticException 발생!
try 블록 나머지 코드는 즉시 중단됩니다.
②

⑥

④ catch 블록 선택 과정
catch 블록 예외 타입 비교 일치? 실행?
ArithmeticException 발생 예외와 동일 타입 ✅ 일치 출력1 실행
ArrayIndexOutOfBoundsException 다른 RuntimeException 자식 ❌ 불일치 건너뜀
NumberFormatException 다른 RuntimeException 자식 ❌ 불일치 건너뜀
Exception 상위 타입 (이미 처리됨) ❌ 도달 안 함 건너뜀
finally 항상 실행 (조건 없음) ✅ 무조건 출력5 실행
⑤ finally 실행 규칙 정리
⑥ 핵심 주의 포인트
  • ➗ int 나눗셈에서 0으로 나누면 ArithmeticException이 발생합니다. (double이면 Infinity가 됩니다.)
  • 📋 catch 블록은 위에서 아래로 순서대로 타입 비교합니다. 첫 번째로 일치한 catch 하나만 실행됩니다.
  • 🏁 finally는 예외 발생 여부와 무관하게 반드시 실행됩니다. return 문이 있어도 finally가 먼저 실행됩니다.
  • 🖨️ 최종 출력: "출력1출력5" (공백 없음, print이므로 줄바꿈 없음)
