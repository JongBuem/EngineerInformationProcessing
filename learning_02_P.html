규칙슬라이싱 핵심 규칙
패턴 의미 예시 (a="hello")
a[start:end] start 포함, end 미포함 a[1:3] → "el"
a[:end] 처음~end-1 a[:3] → "hel"
a[start:] start~끝 a[2:] → "llo"
a[:] 전체 복사 a[:] → "hello"
a[::2] 2칸씩 (짝수 인덱스) a[::2] → "hlo"
a[::-1] 역순 전체 a[::-1] → "olleh"
a[-n:] 뒤에서 n개 a[-2:] → "lo"

⚠ end는 항상 미포함! a[2:5]는 인덱스 2,3,4만 포함합니다. 5는 포함되지 않습니다.

2enumerate() — 인덱스 + 값 동시 순회

enumerate(iterable, start=0): 반복 시 (인덱스, 값) 튜플을 함께 반환합니다. start로 시작 번호를 바꿀 수 있습니다.

Python 
fruits = ['apple', 'banana', 'cherry']

# 기본 사용법
for i, v in enumerate(fruits):
    print(i, v)
# 0 apple
# 1 banana
# 2 cherry

# start 파라미터로 시작 번호 변경
for i, v in enumerate(fruits, start=1):
    print(i, v)
# 1 apple
# 2 banana
# 3 cherry

# dict 컴프리헨션과 결합 (시험 자주 출제)
d = {i: v for i, v in enumerate(fruits)}
# {0: 'apple', 1: 'banana', 2: 'cherry'}

# list()로 변환하면 (index, value) 튜플 리스트
list(enumerate(fruits))
# [(0, 'apple'), (1, 'banana'), (2, 'cherry')]

⚠ enumerate는 원본 순서를 그대로 유지합니다. set처럼 순서가 없는 자료구조에 적용하면 인덱스와 원소 매핑이 실행마다 달라질 수 있습니다.

3set 메서드 총정리 ★ 시험 최빈출

set 메서드는 중복 처리 로직이 핵심입니다. add/remove/update 각 메서드의 동작과 중복 시 동작을 정확히 암기하세요.

메서드 동작 이미 존재 시 없을 때
add(x) 원소 1개 추가 무시 (에러 없음) 추가됨
remove(x) 원소 1개 삭제 — ❌ KeyError 발생
discard(x) 원소 1개 안전 삭제 — 무시 (에러 없음) ✅
update(S) 집합 S 원소 전부 추가 중복 무시 새 원소 추가
pop() 임의 원소 제거 & 반환 — KeyError
clear() 모든 원소 제거 — —
Python — 시험 패턴 완전 추적 
a = {'한국', '중국', '일본'}   # 초기: 3개
a.add('베트남')                  # 추가 → 4개
a.add('중국')                    # 이미 있음 → 무시, 4개 유지
a.remove('일본')                 # 제거 → 3개
a.update({'홍콩', '한국', '태국'}) # 홍콩·태국 추가, 한국 무시 → 5개
print(a)
# {'한국', '중국', '베트남', '홍콩', '태국'} (순서 불정)
★집합 연산 빠른 참조
Python 
a = {1,2,3}; b = {2,3,4}
a | b     # {1,2,3,4}  합집합
a & b     # {2,3}      교집합 ← intersection
a - b     # {1}        차집합 (a에만 있는 것)
a ^ b     # {1,4}      대칭 차집합 (둘 중 하나에만)
a <= b    # False      부분집합 여부
2 in a    # True       멤버십 검사 (set은 O(1)로 빠름)
4type() & isinstance() — 타입 비교

type(x)는 x의 정확한 타입을 반환합니다. isinstance(x, T)는 상속 관계까지 포함해서 확인합니다.

Python 
# type() — 정확한 타입 반환
type(42)         # <class 'int'>
type('hello')   # <class 'str'>
type([1,2])      # <class 'list'>
type(3.14)       # <class 'float'>

# type() == 비교 (시험 출제 패턴)
type(42) == int      # True
type(42) == float    # False
type(42) == str      # False

# isinstance() — 상속 포함 확인
isinstance(42, int)      # True
isinstance(42, (int, float))  # True (여러 타입 중 하나)

# 분기 처리 패턴 (시험 자주 출제)
items = [1, 'a', 2.5, 'b', 3]
for v in items:
    if type(v) == int:
        print('int:', v)
    elif type(v) == str:
        print('str:', v)
값 type() isinstance(v, int)
42 int True
3.14 float False
True bool True (bool은 int의 서브클래스!)
'hello' str False
[1,2] list False

⚠ bool은 int의 서브클래스! isinstance(True, int)는 True입니다. 하지만 type(True) == int는 False입니다. 시험에서 자주 혼동됩니다.

5자주 나오는 함정 포인트
  • ⭐슬라이싱: end는 항상 미포함. a[2:5] → 인덱스 2,3,4만 선택
  • ⭐슬라이싱 범위 초과: a[100:]처럼 범위 벗어나도 에러 없음, 빈 문자열/리스트 반환
  • ⭐set.remove() → 없으면 KeyError / set.discard() → 없어도 안전
  • ⭐set.update({'a','b','c'})의 중복 원소는 자동 무시됨
  • ⭐enumerate 기본 시작값은 0. start=1로 바꾸면 1부터 시작
  • ⭐set은 출력 순서가 매 실행마다 다를 수 있음 — 원소 자체는 동일
  • ⭐type(True) == bool → True / isinstance(True, int) → True (bool은 int 서브클래스)
