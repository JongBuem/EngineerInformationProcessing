Java 추상 클래스 상속 & 메서드 오버로딩 출력 추적

Java · abstract class · extends · 오버로딩 · super · 동적 디스패치 · Tracing
Q문제 코드
Java 
abstract class Vehicle {
    String name;
    abstract public String getName(String val);  // 추상 메서드
    public String getName() {                    // ★ 인자 없는 구현 메서드
        return "Vehicle name: " + name;
    }
}

class Car extends Vehicle {
    public Car(String val) {
        name = super.name = val;  // Vehicle.name = Car.name = "Spark"
    }
    public String getName(String val) {    // 오버라이딩
        return "Car name:" + val;
    }
    public String getName(byte val[]) {     // 오버로딩 (byte[])
        return "Car name:" + val;
    }
}

public class Main {
    public static void main(String[] args) {
        Vehicle obj = new Car("Spark");  // 참조타입: Vehicle
        System.out.println(obj.getName());  // ★ 인자 없는 호출
    }
}

위 코드의 출력값은 무엇인가?

System.out.println(obj.getName()) 출력 결과
Vehicle name: Spark
obj.getName() → 인자 없음 → Vehicle의 getName() 실행
return "Vehicle name: " + name → name = "Spark"
① 문제 유형

추상 클래스 상속, 메서드 오버라이딩/오버로딩, super 필드 초기화에서 올바른 메서드를 선택하는 추적 문제입니다. 참조 타입이 Vehicle이어도 어떤 getName()이 호출되는지 구별하는 것이 핵심입니다.

② 클래스 구조
abstract Vehicle
String name
getName(String val) abstract
getName() 구현됨 ★
extends ↑
concrete Car
getName(String val) override
getName(byte[] val) new overload
getName() → 상속됨 (Vehicle)
③ 핵심 쟁점 — 어떤 getName()이 호출되나?
obj.getName() ← 인자 없음!

Car.getName(String val) → 인자 String 필요 ❌
Car.getName(byte[] val) → 인자 byte[] 필요 ❌
Vehicle.getName()    → 인자 없음 ✅ ← Car에 없어서 상속받은 메서드 실행

∴ Vehicle.getName() 실행
④ 실행 흐름 단계별 추적
1
Vehicle obj = new Car("Spark")
Car 생성자 호출 → name = super.name = "Spark"
2
name = super.name = "Spark"
Vehicle.name = "Spark" 로 설정됨
3
obj.getName() 호출
인자 없음 → Car에 해당 메서드 없음 → Vehicle에서 상속
4
return "Vehicle name: " + name
→ "Vehicle name: Spark"
⑤ getName() 후보 비교표
메서드 위치 시그니처 호출 가능?
getName() Vehicle 인자 없음 ✅ 호출됨 (Car에 상속)
getName(String) Car String val 필요 ❌ 인자 불일치
getName(byte[]) Car byte[] val 필요 ❌ 인자 불일치
getName(String) Vehicle abstract, String val ❌ 인자 불일치
⑥ 핵심 포인트
  • 🔍 obj.getName()에서 인자가 없으므로 매개변수 없는 버전만 호출 후보입니다. 해당 메서드는 Vehicle에 정의되어 있고 Car가 상속받습니다.
  • 🔗 name = super.name = val은 Vehicle의 name 필드를 직접 초기화합니다. 따라서 Vehicle.getName()에서 name="Spark"가 참조됩니다.
  • 📦 참조 타입이 Vehicle이어도 실제 인스턴스는 Car이지만, getName()은 Car가 오버라이딩하지 않았으므로 Vehicle의 구현이 실행됩니다.
