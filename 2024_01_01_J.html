Algorithm Study Note

Java 싱글톤 패턴 + 공유 인스턴스 카운트 추적

Java · Singleton Pattern · static · OOP · Instance · Tracing
Q문제 코드
Java 
class Connection {

    private static Connection _inst = null;  // 유일한 인스턴스
    private int count = 0;

    static public Connection get() {
        if (_inst == null) {
            _inst = new Connection();  // 최초 1회만 생성
            return _inst;
        }
        return _inst;               // 이후엔 동일 인스턴스 반환
    }

    public void count()    { count++; }
    public int  getCount() { return count; }
}

public class main {
    public static void main(String[] args) {

        Connection conn1 = Connection.get();  // 최초 생성 → _inst
        conn1.count();                       // count = 1

        Connection conn2 = Connection.get();  // 동일 인스턴스 반환
        conn2.count();                       // count = 2

        Connection conn3 = Connection.get();  // 동일 인스턴스 반환
        conn3.count();                       // count = 3

        conn1.count();                       // count = 4
        System.out.print(conn1.getCount()); // ?
    }
}

위 코드의 출력값은 무엇인가?

System.out.print() 출력 결과
4
conn1 = conn2 = conn3 = 동일 인스턴스 → count() 총 4회 누적
① 문제 유형

싱글톤 패턴(Singleton Pattern)의 동작을 이해하고, 여러 변수가 동일한 인스턴스를 참조할 때 상태가 누적되는 것을 추적하는 문제입니다.


  • 🔷static Connection _inst — 클래스 전체에서 단 하나만 존재하는 인스턴스 저장소입니다.
  • ⚠️핵심 함정: conn1, conn2, conn3은 서로 다른 변수처럼 보이지만 모두 같은 인스턴스를 가리킵니다. 어느 변수로 count()를 불러도 동일한 count 필드가 증가합니다.

⑤ 싱글톤 패턴(Singleton Pattern) 핵심 원리
요소 역할
private static Connection _inst 단 하나의 인스턴스를 저장하는 클래스 변수
if (_inst == null) new Connection() 최초 1회만 인스턴스 생성 (이후 생성 차단)
return _inst 항상 동일한 객체를 반환 → 모든 변수가 같은 인스턴스 공유
private int count 공유 인스턴스의 상태 — 누가 호출해도 같은 필드 증가
⑥ 핵심 주의 포인트
  • 🔗 conn1 = conn2 = conn3 = 같은 인스턴스 — Connection.get()은 항상 동일한 _inst를 반환합니다.
  • 🔢 count() 총 호출 횟수: conn1(1회) + conn2(1회) + conn3(1회) + conn1(1회) = 4회. 모두 같은 count 필드를 증가시킵니다.
  • 💡 싱글톤 패턴은 프로그램 전체에서 단 하나의 객체만 사용하고 싶을 때 씁니다. DB 연결, 설정 관리 등에 자주 활용됩니다.
