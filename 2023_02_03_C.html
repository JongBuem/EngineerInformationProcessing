Algorithm Study Note

C 전역 배열 포인터 반환 — 같은 주소 덮어쓰기 추적

C · global array · pointer aliasing · gets() · Tracing
Q문제 코드
C 
#include <stdio.h>
#include <stdlib.h>

char n[30];                 // ★ 전역 배열 — 하나만 존재!
char *test() {
    printf("입력하세요 : ");
    gets(n);                 // 항상 같은 전역 n에 씀
    return n;               // 항상 같은 주소 반환
}

int main() {
    char *test1;
    char *test2;
    char *test3;

    test1 = test();          // n ← "홍길동",  test1 = &n
    test2 = test();          // n ← "김철수",  test2 = &n
    test3 = test();          // n ← "박영희",  test3 = &n

    printf("%s\n", test1);   // test1 → n = ?
    printf("%s\n", test2);   // test2 → n = ?
    printf("%s",  test3);   // test3 → n = ?
}

입력값은 홍길동, 김철수, 박영희 순서로 주어진다.

① 문제 유형

전역 배열을 함수가 매번 덮어쓰고 같은 주소를 반환하여, 세 포인터가 모두 동일한 메모리를 가리키는 포인터 앨리어싱(Pointer Aliasing) 함정 문제입니다.

② 핵심 원인 — 전역 배열은 단 하나!
char n[30];  ←  전역 선언 = 프로그램 전체에 딱 1개만 존재

test() 호출마다:
  ① n 배열에 새 입력을 덮어씀
  ② 항상 같은 n의 주소를 반환

결과: test1 == test2 == test3 == &n[0]

⑤ 핵심 주의 포인트
  • ⚠️ 전역 배열은 프로그램에 단 하나만 존재합니다. 세 번의 test() 호출이 모두 같은 n에 덮어쓰므로 마지막 입력만 남습니다.
  • 🔗 test1 = test2 = test3 = &n[0] — 세 포인터 모두 동일한 주소를 가리킵니다.
  • 🖨️ 출력 시점에 n = "박영희" → 세 printf 모두 "박영희" 출력
  • 💡 각각 다른 값을 보존하려면 malloc()이나 지역 배열을 써서 서로 다른 메모리 공간을 확보해야 합니다.
