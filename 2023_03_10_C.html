C 문자열 포인터 산술 & %s / %c 혼합 출력 추적

C · char* · pointer arithmetic · %s · %c · ASCII · Tracing
Q문제 코드
C 
int main() {
    char* p = "KOREA";
    printf("%s\n", p);           // ①
    printf("%s\n", p + 1);       // ② ← 수정됨 (p+3 → p+1)
    printf("%c\n", *p);           // ③
    printf("%c\n", *(p + 3));    // ④
    printf("%c", *p + 4);        // ⑤
}

① 문제 유형

포인터 산술(p+n)과 역참조(*p), %s/%c 형식 지정자의 차이를 묻는 문자열 포인터 추적 문제입니다. 이전 문제에서 p+3이 p+1로 수정되었습니다.


  • 📌%s: 해당 포인터 위치부터 \0까지 문자열 전체 출력
  • 🔤%c: 해당 포인터 위치의 딱 한 글자만 출력
  • ⚠️*p + 4 ≠ *(p+4) — 역참조 후 정수 덧셈 vs 포인터 이동 후 역참조

④ 전체 출력 추적표
# 형식 인수 풀이 출력
① %s p p[0]~p[4] 전체 KOREA
② %s p+1 수정 p[1]~p[4] → 'O'부터 OREA
③ %c *p p[0] = 'K' 한 글자 K
④ %c *(p+3) p[3] = 'E' 한 글자 E
⑤ %c *p+4 'K'(75) + 4 = 79 = 'O' O
⑤ 핵심 주의 포인트
  • 🔴 p+1은 배열 인덱스 [1]인 'O'를 가리키므로, %s로 출력하면 "OREA"가 됩니다.
  • ⚠️ *p + 4 ≠ *(p+4) — *가 먼저 실행되어 'K'(75)를 꺼낸 뒤 4를 더해 79('O')가 됩니다.
  • 📋 최종 출력 5줄: KOREA / OREA / K / E / O
