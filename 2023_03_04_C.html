Algorithm Study Note

C 완전수(Perfect Number) 탐색 — 2~100 합산

C · Perfect Number · 약수 합산 · for loop · Tracing
Q문제 코드
C 
#include <stdio.h>

int test(int n) {
    int i, sum = 0;
    for (i = 1; i <= n / 2; i++) {   // 1 ~ n/2 범위에서 약수 탐색
        if (n % i == 0)
            sum += i;               // 약수이면 합산
    }
    if (n == sum)                   // 자기 자신 = 약수 합 → 완전수
        return 1;
    return 0;
}

int main() {
    int i, sum = 0;
    for (i = 2; i <= 100; i++) {   // 2~100 순회
        if (test(i))
            sum += i;               // 완전수이면 누적
    }
    printf("%d ", sum);
    return 0;
}

위 코드의 출력값은 무엇인가?

printf() 출력 결과
34
2~100 사이의 완전수: 6 + 28 = 34
① 문제 유형

완전수(Perfect Number)를 판별하는 함수를 이용해 2~100 범위에서 완전수를 모두 찾아 합산하는 문제입니다.


  • 📐완전수(Perfect Number)란, 자기 자신을 제외한 양의 약수들의 합이 자기 자신과 같은 수입니다.
  • 🔍test(n)은 i = 1 ~ n/2 범위에서 약수를 합산합니다. n 자체는 포함하지 않으므로 진약수(proper divisor) 합을 구합니다.
  • ✅2~100 사이의 완전수는 6과 28 두 개뿐입니다.
② test() 함수 동작 원리
for (i = 1; i <= n/2; i++) ← 왜 n/2까지만?
n의 약수 중 자기 자신을 제외하면 가장 큰 약수는 최대 n/2입니다.
(예: 28의 진약수 중 최대 = 14 = 28/2)
따라서 i <= n/2로 범위를 제한해도 모든 진약수를 탐색할 수 있습니다.
③ test(6) — 완전수 검증
6 의 진약수 탐색 (i = 1 ~ 3) ✓ 완전수
1 ✓ 2 ✓ 3 ✓
6 % 1 = 0 → sum += 1    sum = 1
6 % 2 = 0 → sum += 2    sum = 3
6 % 3 = 0 → sum += 3    sum = 6
n(6) == sum(6) → return 1 ✓
④ test(28) — 완전수 검증
28 의 진약수 탐색 (i = 1 ~ 14) ✓ 완전수
1 ✓ 2 3 4 ✓ 5 6 7 ✓ 8~13 14 ✓
약수: 1, 2, 4, 7, 14
sum = 1 + 2 + 4 + 7 + 14 = 28
n(28) == sum(28) → return 1 ✓
⑤ 2~100 범위 완전수 탐색 결과
n 진약수 합 완전수? sum 누적
2~5 각각 1, 1, 1+3=4(아님), 1 ✗ 0
6 1 + 2 + 3 = 6 ✓ 완전수 6
7~27 각각 불일치 ✗ 6
28 1+2+4+7+14 = 28 ✓ 완전수 34
29~100 각각 불일치 ✗ 34
최종 sum 34
⑥ 완전수(Perfect Number) 개념
알려진 완전수 목록
6 = 1 + 2 + 3
28 = 1 + 2 + 4 + 7 + 14

8128 = ...
※ 2~100 범위에는 6과 28만 존재합니다.
⑦ 핵심 주의 포인트
  • 🔢 i <= n/2 조건으로 n 자체를 제외한 진약수(proper divisor)만 합산합니다. n 자체를 포함하면 항상 sum > n이 되어 어떤 수도 완전수가 되지 않습니다.
  • 📌 2~100 범위에서 완전수는 6과 28 단 두 개뿐이므로 합은 6 + 28 = 34입니다.
  • 🖨️ 최종 출력: 34 (뒤에 공백 포함: "34 ")
