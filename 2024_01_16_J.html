Java 다형성 — 동적 디스패치(오버라이딩) 추적

Java · Polymorphism · Dynamic Dispatch · Method Overriding · super() · Tracing
Q문제 코드
Java 
class classOne {
    int a, b;

    public classOne(int a, int b) {
        this.a = a;
        this.b = b;
    }

    public void print() {
        System.out.println(a + b);     // classOne의 print()
    }
}

class classTwo extends classOne {
    int po = 3;

    public classTwo(int i) {
        super(i, i + 1);              // super(10, 11)
    }

    public void print() {             // classOne.print() 오버라이딩
        System.out.println(po * po);  // 3 * 3
    }
}

public class main {
    public static void main(String[] args) {
        classOne one = new classTwo(10); // 참조 타입: classOne, 실제 타입: classTwo
        one.print();                    // 어떤 print()가 호출될까?
    }
}

위 코드의 출력값은 무엇인가?

System.out.println() 출력 결과
9
참조 타입은 classOne이지만 실제 객체는 classTwo
→ 동적 디스패치 → classTwo.print() 실행 → po*po = 3*3 = 9
① 문제 유형

참조 변수의 타입(classOne)과 실제 객체의 타입(classTwo)이 다를 때, 오버라이딩된 메서드 중 어느 것이 호출되는지를 추적하는 다형성(Polymorphism) 문제입니다.


  • ⚠️핵심 함정: classOne one = new classTwo(10) — 변수 타입이 classOne이어도 실제 객체가 classTwo이면 classTwo의 print()가 호출됩니다.
  • 🔄동적 디스패치(Dynamic Dispatch): Java는 오버라이딩된 메서드를 실행 시점에 실제 객체 타입 기준으로 결정합니다.
  • 🖨️classTwo의 print()는 po * po = 3 * 3 = 9를 출력합니다.

Java의 동적 디스패치: 오버라이딩된 메서드는
참조 변수의 타입이 아닌 실제 객체의 타입을 기준으로 호출됩니다.
→ one이 classOne 타입으로 선언되었어도, 객체가 classTwo이므로 classTwo.print() 실행

⑤ 오버라이딩 vs 오버로딩 — 이전 문제와 비교
이 문제 — 오버라이딩 ✓
classOne.print() // 파라미터 없음
classTwo.print() // 파라미터 없음
→ 시그니처 동일 = 오버라이딩
→ 동적 디스패치 발생
이전 문제 — 오버로딩 ✗
Parent.getT() // 파라미터 없음
Child.getT(int n) // 파라미터 있음
→ 시그니처 다름 = 오버로딩
→ 동적 디스패치 없음
⑥ 핵심 주의 포인트
  • 🎯 오버라이딩 + 다형성: 부모 타입 참조 변수로 자식 객체를 가리킬 때, 오버라이딩된 메서드는 항상 자식 버전이 호출됩니다.
  • 🔢 classTwo.print() 실행 → po * po = 3 * 3 = 9
  • 📌 만약 classOne one = new classOne(10, 11)이었다면 a + b = 21이 출력되었을 것입니다.
