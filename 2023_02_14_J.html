Java String — 리터럴 vs new String() 참조 비교 추적

Java · String Pool · Heap · == (참조비교) · equals() (값비교) · Tracing
Q문제 코드
Java 
public class Main {
    public static void main(String[] args) {
        String str1 = "Programming";         // String Pool
        String str2 = "Programming";         // String Pool — str1과 같은 객체!
        String str3 = new String("Programming"); // Heap — 새 객체!

        println(str1 == str2);          // ① 참조 비교
        println(str1 == str3);          // ② 참조 비교
        println(str1.equals(str3));     // ③ 값 비교
        print(str2.equals(str3));      // ④ 값 비교 (줄바꿈 없음)
    }
}

위 코드의 출력값은 무엇인가?

리터럴 → String Pool 공유 → == true

① 문제 유형

Java의 String Pool과 Heap 메모리 차이를 이해하고, == (참조 비교)와 equals() (값 비교)의 결과를 추적하는 문제입니다.

② 메모리 구조 시각화
Stack (변수)
str1
──────┐
str2
──────┤ 동일 주소
str3
──────→ Heap
String Pool (Heap 내)
0x1000
"Programming"
← str1, str2 둘 다 이 주소
Heap (new)
0x2000
"Programming"
← str3 만 이 주소
값은 같지만 주소가 다름!
str1 주소 = 0x1000  |  str2 주소 = 0x1000  |  str3 주소 = 0x2000
str1 == str2 → 0x1000 == 0x1000 → true
str1 == str3 → 0x1000 == 0x2000 → false
③ 단계별 출력 추적표
# 코드 비교 방식 이유 결과
① str1 == str2 참조 비교 둘 다 String Pool의 같은 주소 0x1000 true
② str1 == str3 참조 비교 str1=0x1000 (Pool) vs str3=0x2000 (Heap) → 다른 주소 false
③ str1.equals(str3) 값 비교 내용 "Programming" == "Programming" → 같은 값 true
④ str2.equals(str3) 값 비교 내용 "Programming" == "Programming" → 같은 값 true
④ == vs equals() 핵심 차이
==       → 주소(참조) 비교   // 같은 객체인지 확인
.equals() → 내용(값) 비교    // 문자열 내용이 같은지 확인
⑤ String Pool vs new String()
  • 🏊 리터럴 선언 ("...") → JVM이 String Pool에서 같은 문자열 재사용 → str1, str2는 동일 객체
  • 🆕 new String("...") → 항상 Heap에 새 객체 생성 → 내용이 같아도 주소는 다름
  • ✅ 문자열 내용 비교는 반드시 .equals() 를 사용해야 합니다. ==는 참조 주소를 비교합니다.
