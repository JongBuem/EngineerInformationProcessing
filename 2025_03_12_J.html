Java super() 생성자 호출 빈칸 채우기

Java · OOP · Inheritance · super() · Constructor · Fill-in-the-blank
Q문제 코드 — 빈칸에 들어갈 키워드는?
Java 
class Rectangle {
    int width, height;

    Rectangle(int width, int height) {
        this.width  = width;
        this.height = height;
    }
}

class Square extends Rectangle {

    Square(int a) {
        ____(a, a);     // 부모 생성자에 a, a 전달
    }

    int getSquareArea() {
        return width * height;
    }
}

public class Main {
    public static void main(String[] args) {
        Square sq = new Square(10);
        System.out.println(sq.getSquareArea());
    }
}

빈칸 ____ 에 들어갈 키워드와, 최종 출력값은 무엇인가?

① 문제 유형

자식 클래스 생성자에서 부모 클래스 생성자를 명시적으로 호출하는 super() 키워드를 묻는 빈칸 채우기 문제입니다.


  • 🔷super() — 자식 생성자 내에서 부모 클래스의 생성자를 호출합니다. 반드시 첫 번째 줄에 위치해야 합니다.
  • 🔷상속된 필드 초기화 — Rectangle의 width, height는 super(a, a)를 통해 초기화됩니다. Square가 직접 선언한 필드가 아닙니다.
  • 🔷정사각형 논리 — 가로·세로가 같으므로 super(a, a)로 두 값을 동일하게 전달합니다.
② 정답 적용 코드
Java — 정답 적용 
class Square extends Rectangle {

    Square(int a) {
        super(a, a);    // Rectangle(a, a) 호출 → width=a, height=a
    }

    int getSquareArea() {
        return width * height;   // a × a
    }
}

④ 단계별 실행 추적
단계 실행 구문 width height 설명
① new Square(10) — — Square 생성자 호출, a=10
② super(10, 10) — — Rectangle(10, 10) 생성자 실행
③ this.width = 10 10 — Rectangle 생성자 내부
④ this.height = 10 10 10 Rectangle 생성자 내부
⑤ getSquareArea() 10 10 width * height 계산
출력 println(100) 10 10 10 × 10 = 100
⑤ super() vs this() — 생성자 호출 비교
키워드 용도 호출 대상 위치 제약
super() 부모 클래스 생성자 호출 부모 클래스의 생성자 자식 생성자 첫 번째 줄
this() 같은 클래스 다른 생성자 호출 동일 클래스의 다른 생성자 생성자 첫 번째 줄
⑥ 핵심 정리
  • 📌자식 클래스 생성자에서 부모 생성자를 명시적으로 호출할 때는 super(인수)를 사용합니다.
  • 🏗️Square는 width, height 필드를 직접 선언하지 않습니다. 상속받은 부모의 필드를 super(a, a)로 초기화합니다.
  • ⚠️super()는 반드시 생성자의 첫 번째 줄에 위치해야 합니다. 위치가 틀리면 컴파일 오류가 발생합니다.
  • 🖨️최종 출력: getSquareArea() = width × height = 10 × 10 = 100
