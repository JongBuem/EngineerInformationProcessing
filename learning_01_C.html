1* — 선언과 사용, 두 가지 의미
1선언에서의 * — 포인터 변수 선언
C 
int* a;            // a는 int가 저장된 메모리 주소를 들고 있는 변수
struct node* p;    // p는 struct node를 가리키는 포인터
char* s;           // s는 char를 가리키는 포인터
  • 📌a는 int 값 자체가 아니라, int가 저장된 메모리 주소를 들고 있는 변수입니다.
  • 📌p는 struct node 구조체를 가리키는 포인터 — 구조체 자체가 아닌 주소를 보관합니다.
  • 📌s는 char 한 글자를 가리키는 포인터 — 문자열의 현재 위치(주소)를 들고 있습니다.
2사용에서의 * — 역참조, 실제 값 접근
C 
while (*s)   // s가 가리키는 실제 문자가 '\0'이 아닌 동안 반복
  • 🔷s = 현재 문자의 주소
  • 🔷*s = s가 가리키는 곳에 들어 있는 실제 문자

2-> — 구조체 포인터의 멤버 접근

구조체 변수는 .으로 멤버에 바로 접근 가능합니다:

C 
struct node x;
x.c = 'A';   // 구조체 변수 직접 접근 → . 사용 가능

그러나 포인터는 주소를 들고 있으므로 .을 바로 쓸 수 없습니다:

C 
struct node* n;   // n은 구조체 자체가 아니라 주소
n->c              // 포인터를 통한 멤버 접근

n->c는 아래와 완전히 동일합니다:

n->c
축약 표현
≡
(*n).c
풀어쓴 표현

동작 순서 분해:

①
*n
n이 가리키는 실제 구조체를 꺼냄 (역참조)
②
(*n).c
그 구조체의 c 멤버에 접근
③
n->c
① + ② 두 단계를 화살표 연산자 하나로 축약한 것
상황 사용할 연산자 예시
구조체 변수의 멤버 접근 . x.val
구조체 포인터의 멤버 접근 -> (= (*ptr).field) n->val
3& — 변수의 주소를 가져오는 연산자
C 
int a = 10;
int* p = &a;   // p에 a의 주소를 저장
a = 10

⟵

표현식 의미 값
a 변수 자체 10
&a a의 메모리 주소 0x100 (예시)
p 주소를 저장한 포인터 변수 0x100
*p p가 가리키는 실제 값 (역참조) 10

전체 흐름 요약:

C 
a        // 값 (10)
&a       // a의 주소 (0x100)
p = &a   // p에 주소 저장
*p       // p가 가리키는 실제 값 → 10
4세 연산자 연계 예시
C 
struct node {
    int val;
    struct node* next;
};

struct node a = {10, 0};
struct node* p = &a;   // p에 a의 주소 저장  ← & 사용

p->val   // = (*p).val = a.val = 10  ← -> 사용
*p       // 구조체 a 자체            ← * 역참조
&a       // a의 주소 (p와 동일한 값) ← & 사용
5한 줄 요약
  • ⭐ * 선언 : "이 변수는 주소를 저장한다"
  • ⭐ * 사용 : "이 주소가 가리키는 실제 값을 꺼낸다" (역참조)
  • ⭐ -> : "포인터가 가리키는 구조체의 멤버를 꺼낸다" — (*p).field의 축약
  • ⭐ & : "이 변수가 저장된 메모리 주소를 알려준다"
C — 세 연산자 한눈에 보기 
int a = 10;
int* p = &a;   // & : a의 주소 가져오기 / * : 포인터 선언
*p = 20;      // * : p가 가리키는 실제 값 변경 → a = 20

struct node x = {42, 0};
struct node* q = &x;   // & : x의 주소 / * : 포인터 선언
q->val;                 // -> : 포인터로 구조체 멤버 접근 → 42
// q->val  ≡  (*q).val  ≡  x.val
