Java enum 메서드 연쇄 추적

Java · enum · values() · name() · length() · Tracing
Q문제 코드
Java 
enum Tri {
    A("A"), B("AB"), C("ABC");

    private String code;

    Tri(String code) {
        this.code = code;
    }

    public String code() {
        return code;
    }
}

public class Main {
    public static void main(String[] args) {
        Tri t = Tri.values()[Tri.A.name().length()];
        System.out.print(t.code());
    }
}

위 코드의 출력값은 무엇인가?

System.out.print() 출력 결과
AB
Tri.A.name()="A" → length()=1 → values()[1]=B → B.code()="AB"
① 문제 유형

Java enum 내장 메서드(values, name)와 String 메서드(length)를 연쇄 호출하여 출력값을 추적하는 문제입니다.


  • 🔷values() — enum의 모든 상수를 선언 순서대로 배열로 반환합니다.
  • 🔷name() — enum 상수의 이름 문자열을 반환합니다. (소스코드에 선언된 이름 그대로)
  • 🔷code() — 이 enum이 직접 정의한 메서드로, 생성자에서 전달된 code 필드 값을 반환합니다.
② enum Tri 구조
③ 연쇄 호출 단계별 분해
1 Tri.A.name() → "A" enum 상수 A의 이름 문자열
2 "A".length() → 1 문자열 "A"의 길이 = 1
3 Tri.values()[1] → B 인덱스 1번 = 두 번째 상수 B
4 t = B → Tri.B t에 enum 상수 B 할당
5 t.code() → "AB" B의 code 필드값 반환 → 출력!
④ 단계별 추적 테이블
단계 호출 반환 타입 값 근거
① Tri.A.name() String "A" enum 상수 이름 그대로 반환
② "A".length() int 1 "A"는 한 글자 → 길이 1
③ Tri.values()[1] Tri B values() = {A,B,C}, 인덱스 1 = B
④ t = B Tri Tri.B 변수 t에 B 할당
⑤ t.code() String "AB" B 생성자에 전달된 code = "AB"
⑤ 자주 쓰이는 enum 내장 메서드
메서드 반환 타입 설명 이 코드에서 결과
values() Tri[] 모든 상수를 배열로 반환 (선언 순서) {A, B, C}
name() String 상수의 이름 문자열 반환 A.name() = "A"
ordinal() int 선언 순서 인덱스 반환 (0부터) A=0, B=1, C=2
code() String 사용자 정의 메서드 (code 필드 반환) B.code() = "AB"
⑥ 핵심 주의 포인트
  • 📌 name()은 소스코드에 선언된 상수 이름 그대로 반환합니다. Tri.A.name() → "A" (대문자 그대로)
  • 🔢 "A".length() = 1 — 이 값이 values() 배열의 인덱스로 사용됩니다. 인덱스 1 = 두 번째 원소 B
  • ⚠️ code()와 name()을 혼동하지 마세요. name()은 "B", code()는 생성자에 전달된 "AB"를 반환합니다.
  • 🖨️ System.out.print (println 아님) — 줄바꿈 없이 AB만 출력합니다.
