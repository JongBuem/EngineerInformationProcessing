C 지역 변수 스코프 + switch 폴스루(Fall-through)

C · Local Scope · Function · switch · Fall-through · break · Tracing
Q문제 코드
C 
#include <stdio.h>

void swap() {
    int a = 11;
    int b = 19;
    int t = a;
    a = b;              // 지역변수만 바뀜!
    b = t;              // main의 a, b와 무관
}

int main() {
    int a = 11;
    int b = 19;
    swap();              // main의 a, b는 그대로!
    switch(a) {         // a = 11
        case 1:
            b += 1;      // 건너뜀
        case 11:           // ← 매칭!
            b += 2;      // b = 21
        default:           // break 없어 폴스루!
            b += 3;      // b = 24
            break;
    }
    printf("%d", a - b); // 11 - 24 = -13
}

위 코드의 출력값은 무엇인가?

printf() 출력 결과
-13
a=11, b=19+2+3=24 → 11−24 = −13
① 문제 유형

C 함수의 지역 변수 스코프와 switch 문의 폴스루(Fall-through) 동작을 동시에 이해하는 문제입니다.


  • ⚠️함정 ①: swap()은 인자 없이 자신만의 지역 변수를 사용합니다. main의 a, b에 아무런 영향을 주지 않습니다.
  • ⚠️함정 ②: case 11:에 break가 없으므로 아래 default:까지 폴스루(fall-through)됩니다.
② 스택 프레임 — swap()은 독립적
main() 스택 프레임
a11
b19 → 24
swap() 호출 전후 변화 없음
↔
swap() 스택 프레임
a11 → 19
b19 → 11
t11

swap() 내부 a, b는 완전히 독립된 지역 변수 — 리턴 후 스택에서 소멸

③ switch(a=11) 실행 흐름 — 폴스루 주의!
case 1: b += 1 ❌ a≠1, 건너뜀
case 11: b += 2  → b = 19+2 = 21 ✅ 진입
↓ break 없음 → 폴스루!
default: b += 3  → b = 21+3 = 24 ↓ 폴스루
break; switch 탈출 ⛔ 종료
④ 단계별 변수 상태 추적
단계 이벤트 a b 비고
1 선언 초기화 11 19 int a=11, b=19
2 swap() 호출 11 (불변) 19 (불변) 지역 스코프 — main 무영향
3 case 11: b+=2 11 21 19 + 2 = 21
4 default: b+=3 (폴스루) 11 24 21 + 3 = 24
5 break — switch 탈출 11 24
6 printf("%d", a-b) 11 24 11 − 24 = −13
a  =  11  (변화 없음)
b  = 19 + 2(case 11) + 3(default) = 24
a − b = 11 − 24 = −13
⑤ 핵심 주의 포인트
  • 🔒 C 지역 변수 스코프: swap()은 포인터를 전달받지 않았으므로 main의 변수에 접근 불가. 반환 후 swap의 모든 변수는 소멸됩니다.
  • ⬇️ switch 폴스루(Fall-through): 매칭된 case 이후 break나 return이 없으면 다음 case/default 코드가 조건 없이 연속 실행됩니다.
  • 📌 case 1:은 a=11이므로 건너뛰고, case 11:에서 진입 후 break 없이 default:까지 실행됩니다.
  • 🖨️ 최종 출력: -13
