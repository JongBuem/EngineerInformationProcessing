Java 에러 발생 라인 찾기 — static & 인스턴스 변수

Java · static · instance field · Compile Error · non-static context · Error Line
Q문제 코드 — 에러가 발생하는 라인은?
Java 
1class Person {


2    private String name;


3    public Person(String val) {


4        name = val;


5    }


6    public static String get() {


7        return name;  // ← ?


8    }


9    public void print() {


10        System.out.println(name);


11    }


12}


13public class main {


14    public static void main(String[] args) {


15        Person obj = new Person("Kim");


16        obj.print();


17    }


18}

컴파일 에러가 발생하는 라인 번호와 이유를 찾아라.

에러 발생 라인
Line 7
return name;
non-static variable name
cannot be referenced from a static context

static 메서드 get() 에서 인스턴스 변수 name에 접근 불가
① 문제 유형

static 메서드 안에서 인스턴스(non-static) 변수를 참조하는 위치를 찾는 컴파일 에러 탐색 문제입니다.

② 에러 위치 & 컴파일러 메시지
Line 7:  return name;

error: non-static variable name
       cannot be referenced from a static context

← Line 6의 public static String get() 가 원인
③ 원인 분석 — static vs 인스턴스
Line 6 — static 메서드
public static String get()
클래스 레벨 — 객체 없이 존재
→ 어느 객체의 name?
알 수 없음 ✕
Line 2 — 인스턴스 변수
private String name
객체 레벨 — 객체마다 존재
→ new Person() 후 생성
객체 귀속 ✓
④ 라인별 정상 / 에러 분류
라인 코드 name 접근 판정
4 name = val 생성자(인스턴스 문맥) 정상
7 return name static 메서드 내부 컴파일 에러 ★
10 println(name) 인스턴스 메서드 내부 정상
⑤ 수정 방법
Java — 수정안 
// 방법 1: static 제거 (가장 권장)
public String get() { return name; }

// 방법 2: name도 static으로 변경
private static String name;
⑥ 핵심 규칙
  • 🚫 static 메서드 → 인스턴스 변수 직접 접근 불가 (어느 객체의 변수인지 특정할 수 없음)
  • ✅ 인스턴스 메서드 → static 변수 / 인스턴스 변수 모두 접근 가능
  • 📍 에러 라인: Line 7 — return name; (static get() 내부)
