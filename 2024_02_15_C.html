Algorithm Study Note

C 2차원 배열 + 포인터 배열 역참조 추적

C · 2D Array · Pointer Array · Subscript · Dereference · Arithmetic · Tracing
Q문제 코드
C 
#include <stdio.h>

int main() {
    int arr[3][3] = {1,2,3,4,5,6,7,8,9};
    int* parr[2] = {arr[1], arr[2]};

    printf("%d",
        parr[1][1]          // ① 첨자 접근
      + *(parr[1]+2)      // ② 포인터 산술 + 역참조
      + **parr              // ③ 이중 역참조
    );
    return 0;
}

위 코드의 출력값은 무엇인가?

printf() 출력 결과
21
8 + 9 + 4 = 21
① 문제 유형

2차원 배열의 행 포인터를 저장한 포인터 배열에서, 첨자·포인터 산술·이중 역참조 세 가지 접근법을 동시에 이해하는 문제입니다.


  • 🔷arr[r]은 r행의 첫 번째 원소를 가리키는 int* 포인터입니다.
  • ⚠️세 가지 역참조 표현: parr[1][1](첨자), *(parr[1]+2)(산술), **parr(이중)이 각각 다르게 생겼지만 모두 특정 원소를 가리킵니다.
② arr[3][3] 메모리 배치
arr[0]
1[0][0]
2[0][1]
3[0][2]
← parr에 포함 안 됨
arr[1] → parr[0]
4[1][0]
5[1][1]
6[1][2]
parr[0] = arr[1] → 4,5,6
arr[2] → parr[1]
7[2][0]
8[2][1]
9[2][2]
parr[1] = arr[2] → 7,8,9
⭐ 표시(금테): 계산에 사용되는 원소
③ int* parr[2] 포인터 배열 구조
parr[0]
가리키는 행arr[1]
원소{4, 5, 6}
*parr→ 4 (arr[1][0])
parr[1]
가리키는 행arr[2]
원소{7, 8, 9}
parr[1][0]→ 7
④ 세 항 분해 및 값 추적
①
parr[1][1]
= *(parr[1] + 1)
= parr[1]은 arr[2] → arr[2][1]
→
arr[2][1] = 8
②
*(parr[1] + 2)
= parr[1]은 arr[2] 시작점
= parr[1]+2 → arr[2][2]
→
arr[2][2] = 9
③
**parr
= *parr → parr[0] (= arr[1] 주소)
= *parr[0] → arr[1][0]
→
arr[1][0] = 4
⑤ 단계별 역참조 추적
항 표현식 parr 인덱스 가리키는 배열 최종 원소 값
① parr[1][1] parr[1] = arr[2] arr[2][1] arr[2][1] 8
② *(parr[1]+2) parr[1] = arr[2] arr[2]+2 = arr[2][2] arr[2][2] 9
③ **parr *parr = parr[0] = arr[1] *arr[1] = arr[1][0] arr[1][0] 4
합계 = 8+9+4 = 21
parr[1][1]  +  *(parr[1]+2)  +  **parr
=  8  +  9  +  4  =  21
⑥ 세 가지 역참조 표현 비교
표현 풀어쓰기 의미
parr[i][j] *(parr[i]+j) i행 j열 원소 (첨자 표기)
*(parr[i]+j) parr[i][j] 포인터+오프셋 역참조
**parr *parr[0] = parr[0][0] parr → parr[0] → 첫 원소
⑦ 핵심 주의 포인트
  • 📌 parr[0] = arr[1], parr[1] = arr[2] — arr[0]은 포함되지 않습니다.
  • 🔢 **parr는 *(*(parr+0)) = *parr[0] = parr[0][0] = arr[1][0] = 4입니다.
  • ➕ *(parr[1]+2)에서 +2는 바이트가 아닌 int 단위(4바이트씩) 이동입니다. arr[2][0]에서 2칸 → arr[2][2] = 9.
  • 🖨️ 최종 출력: 8 + 9 + 4 = 21
