C 문자열 포인터 역순 + 홀수 인덱스 출력 추적

C · String · Pointer · reverse() · strlen · for loop · Tracing
Q문제 코드
C 
#include <stdio.h>
#include <string.h>

void reverse(char* str) {
    int len = strlen(str);
    char temp;
    char* p1 = str;
    char* p2 = str + len - 1;   // 마지막 문자 주소
    while (p1 < p2) {
        temp = *p1;
        *p1 = *p2;
        *p2 = temp;
        p1++; p2--;            // 양끝에서 안쪽으로
    }
}

int main(int argc, char* argv[]) {
    char str[100] = "ABCDEFGH";

    reverse(str);              // 문자열 역순

    int len = strlen(str);

    for (int i = 1; i < len; i += 2) {
        printf("%c", str[i]);   // 홀수 인덱스만 출력
    }

    printf("\n");
    return 0;
}

① 문제 유형

포인터를 이용해 문자열을 제자리(in-place)에서 역순으로 뒤집은 뒤, 홀수 인덱스 위치의 문자만 출력하는 2단계 추적 문제입니다.


  • 🔄1단계: reverse()가 p1(앞), p2(뒤) 두 포인터를 안쪽으로 좁히며 문자를 swap. "ABCDEFGH" → "HGFEDCBA"
  • 🔢2단계: i=1부터 i+=2로 홀수 인덱스(1,3,5,7)만 순회하여 출력.
  • ⚠️함정: 역순 이후의 배열을 기준으로 인덱스를 읽어야 합니다. 원본 기준으로 잘못 읽으면 오답.

⑤ 핵심 주의 포인트
  • 🔄 reverse()는 원본 배열을 직접 수정합니다. 포인터를 통해 str을 바꾸므로 main에서도 반영됩니다.
  • 🔢 i=1; i+=2는 홀수 인덱스(1, 3, 5, 7)만 방문합니다. 역순 이후의 배열을 기준으로 읽어야 합니다.
  • 📌 역순 후 배열: [H][G][F][E][D][C][B][A] → 홀수 위치 = G, E, C, A
  • 🖨️ 최종 출력: GECA (뒤에 개행 \n 포함)
