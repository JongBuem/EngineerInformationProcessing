Algorithm Study Note

Java 제네릭 타입 소거 + 오버로딩 해석 추적

Java · Generics · Type Erasure · Overloading · Compile-time · Object · Tracing
Q문제 코드
Java 
class Printer {
    void print(Integer a) { System.out.print("A" + a); }  // ① Integer
    void print(Object  a) { System.out.print("B" + a); }  // ② Object
    void print(Number  a) { System.out.print("C" + a); }  // ③ Number
}

public class Generic {
    public static void main(String[] args) {
        new Container<>(0).print();   // T = Integer로 추론되지만…
    }

    public static class Container<T> {
        T value;
        public Container(T t) { value = t; }
        public void print() {
            new Printer().print(value);  // 컴파일 시 value의 타입은 Object!
        }
    }
}

위 코드의 출력값은 무엇인가?

System.out.print() 출력 결과
B0
T는 런타임에 Object로 소거 → print(Object) 선택 → "B" + 0
① 문제 유형

Java 제네릭의 타입 소거(Type Erasure)로 인해 컴파일 타임에 T가 Object로 처리되어, 오버로딩 해석 시 print(Object)가 선택되는 문제입니다.


  • ⚠️핵심 함정: new Container<>(0)으로 T=Integer가 추론되지만, 제네릭 Container<T> 내부에서 value의 컴파일 타임 타입은 Object입니다.
  • 🔷오버로딩 해석은 컴파일 타임에 결정됩니다. value의 타입이 Object이므로 가장 근접한 print(Object)가 선택됩니다.
  • 🔷Integer → Number → Object 계층에서 print(Integer), print(Number)가 있어도 컴파일러가 보는 타입은 Object이므로 무시됩니다.
② 타입 소거(Type Erasure) — 제네릭의 핵심 원리

컴파일 후 바이트코드에서 T는 사라지고 Object(또는 상한 경계)로 대체됩니다

❌ 착각하기 쉬운 관점 (소스 코드)
Container<Integer> c = new Container<>(0);
c.print();
→ T = Integer이니까 print(Integer)?
✅ 실제 컴파일러 관점 (바이트코드)
Container 내부에서 T → Object
value의 타입 = Object
→ print(Object) 선택!
// 컴파일 후 Container 클래스 (개념적 바이트코드)
class Container {
  Object value; // T → Object 소거
  void print() { new Printer().print((Object) value); }
}
③ Java 타입 계층 — 오버로드 후보 분석

오버로딩 해석 = 컴파일 타임 타입 기준 → value는 Object → print(Object) 선택

④ 단계별 실행 흐름
1
new Container<>(0) 생성
0은 int → autoboxing → Integer(0)
T = Integer로 추론, value = Integer(0) 저장
2
Container.print() 호출
내부에서 T → Object 소거
value의 컴파일 타임 타입 = Object
3
오버로딩 해석 (컴파일 타임)
인수 타입 = Object
→ print(Integer): Object는 Integer 아님 ❌
→ print(Number) : Object는 Number 아님 ❌
→ print(Object) : Object == Object ✅
4
print(Object a) 실행
System.out.print("B" + a)
a = 0 → 출력: B0
⑤ 오버로드 후보 매칭 분석
후보 메서드 파라미터 타입 인수 타입(소거 후) 매칭 여부 출력
print(Integer a) Integer Object ❌ Object → Integer 불가 —
print(Number a) Number Object ❌ Object → Number 불가 —
print(Object a) Object Object ✅ 정확히 일치 B0
⑥ 핵심 주의 포인트
  • 🗑️ 제네릭 타입 소거(Type Erasure): Java 제네릭은 컴파일 후 타입 정보가 사라집니다. 상한 경계가 없는 <T>는 Object로 소거됩니다.
  • ⚖️ 오버로딩 해석은 컴파일 타임: 실제 런타임 값(Integer 0)이 아닌, 컴파일러가 보는 정적 타입(Object)을 기준으로 메서드를 선택합니다.
  • 💡 만약 Container<T extends Number>였다면 T → Number로 소거되어 print(Number)가 선택되고 C0이 출력됩니다.
  • 🖨️ 최종 출력: B0
System.out.print() 출력 결과
B0
T 소거 → Object → print(Object) 선택 → "B" + 0 = B0
