Algorithm Study Note

Java 동적 디스패치 vs 정적 메서드 숨김

Java · OOP · Dynamic Dispatch · Static Hiding · Override · Tracing
Q문제 코드
Java 
public class Main {
    public static class Parent {
        public int x(int i)      { return i + 2; }
        public static String id() { return "P"; }
    }
    public static class Child extends Parent {
        public int x(int i)       { return i + 3; }   // 오버라이드
        public String x(String s) { return s + "R"; } // 오버로드 (별개)
        public static String id()  { return "C"; }   // 숨김(Hiding)
    }
    public static void main(String[] args) {
        Parent ref = new Child();           // 선언: Parent, 실제: Child
        System.out.println(ref.x(2) + ref.id());
    }
}

위 코드의 출력값은 무엇인가?

System.out.println() 출력 결과
5P
ref.x(2) → Child.x(2)=5 (동적) │ ref.id() → Parent.id()="P" (정적 숨김)
① 문제 유형

인스턴스 메서드의 동적 디스패치(Dynamic Dispatch)와 static 메서드의 정적 바인딩(숨김, Hiding)의 차이를 묻는 추적 문제입니다.


  • 🟢인스턴스 메서드 — 실행 시간(Runtime)에 실제 객체 타입을 기준으로 결정됩니다. (동적 디스패치)
  • 🟡static 메서드 — 컴파일 시간에 참조 변수 타입을 기준으로 결정됩니다. (정적 바인딩 / 메서드 숨김)
  • 🔷Parent ref = new Child() — 선언 타입은 Parent, 실제 객체는 Child입니다. 이 차이가 핵심입니다.
② ref.x(2) vs ref.id() — 어느 클래스 메서드가 호출되나?
③ 단계별 실행 추적
호출 메서드 종류 바인딩 기준 실행 메서드 반환값
ref.x(2) 인스턴스 메서드 실제 객체 → Child Child.x(2) = 2+3 5
ref.id() static 메서드 선언 타입 → Parent Parent.id() = "P" "P"
5 + "P" = "5P"
④ Override(오버라이드) vs Hiding(숨김) 비교
구분 대상 결정 시점 기준 이 코드에서
Override 인스턴스 메서드 Runtime 실제 객체 타입 x(int) → Child 버전 실행
Hiding static 메서드 Compile-time 참조 변수 타입 id() → Parent 버전 실행
Overload 같은 이름, 다른 시그니처 Compile-time 매개변수 타입 x(String)는 Parent에 없어 ref로 호출 불가
⑤ 핵심 주의 포인트
  • 📌 Parent ref = new Child() — 선언 타입(Parent)과 실제 객체(Child)가 다릅니다. 이게 핵심 함정입니다.
  • 🟢 ref.x(2) — 인스턴스 메서드는 실제 객체(Child) 기준. Child.x(2) = 2+3 = 5
  • 🟡 ref.id() — static 메서드는 선언 타입(Parent) 기준. Parent.id() = "P" (Child.id()가 아님!)
  • ⚠️ Child.x(String)는 Parent에 없어 ref로 호출할 수 없습니다. 오버로드는 별개의 메서드입니다.
  • 🖨️ 5 + "P" → 정수 5와 문자열 "P"의 + 연산 → 문자열 연결 = "5P"
