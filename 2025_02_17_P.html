Algorithm Study Note

Python 딕셔너리 컴프리헨션 & 집합 교집합 추적

Python · dict comprehension · set · intersection · values() · Tracing
Q문제 코드
Python 
lst = [1, 2, 3]
dst = {i: i * 2 for i in lst}     # 딕셔너리 컴프리헨션
s   = set(dst.values())             # dst 값들로 집합 생성

lst[0] = 99                          # lst 변경 → dst에 영향 없음
dst[2] = 7                           # dst 직접 수정
s.add(99)                           # s에 99 추가

print(len(s & set(dst.values())))

위 코드의 출력값은 무엇인가?

print() 출력 결과
2
s={2,4,6,99} & dst.values()={2,7,6} → 교집합={2,6} → len=2
① 문제 유형

딕셔너리 컴프리헨션 생성 후 독립적인 변수 변경과 집합 교집합 연산 결과를 추적하는 문제입니다.


  • 🔷s = set(dst.values()) — 생성 시점의 스냅샷입니다. 이후 dst가 바뀌어도 s는 자동으로 바뀌지 않습니다.
  • 🔷lst[0] = 99 — dst는 컴프리헨션으로 생성된 별도 딕셔너리입니다. lst 변경은 dst에 영향 없습니다.
  • 🔷s & set(dst.values()) — &는 집합 교집합 연산자입니다.
② 단계별 변수 추적
단계 실행 구문 lst dst s
① lst = [1, 2, 3] [1, 2, 3] — —
② dst = {i: i*2 for i in lst} [1, 2, 3] {1:2, 2:4, 3:6} —
③ s = set(dst.values()) [1, 2, 3] {1:2, 2:4, 3:6} {2, 4, 6}
④ lst[0] = 99 [99, 2, 3] {1:2, 2:4, 3:6} 불변! {2, 4, 6} 불변!
⑤ dst[2] = 7 [99, 2, 3] {1:2, 2:7, 3:6} {2, 4, 6} 불변!
⑥ s.add(99) [99, 2, 3] {1:2, 2:7, 3:6} {2, 4, 6, 99}
⑦ s & set(dst.values()) — values={2,7,6} {2,4,6,99}
③ 교집합(&) 연산 시각화
s =
2 4 6 99
dst.values() =
2 7 6
s & dst =
2 6
← 두 집합 모두에 있는 원소만
len() = 2
■ 교집합(공통)   ■ s에만 있음 (4, 99 탈락)   ■ dst에만 있음 (7 탈락)
④ 핵심 함정 — lst 변경이 dst에 영향 없는 이유
  • 📸 딕셔너리 컴프리헨션 {i: i*2 for i in lst}은 생성 시점에 값을 복사합니다. 이후 lst를 변경해도 dst는 바뀌지 않습니다.
  • 📸 s = set(dst.values())도 마찬가지입니다. 생성 시점의 값들로 독립적인 집합을 만듭니다. 이후 dst가 바뀌어도 s는 자동 반영되지 않습니다.
⑤ 핵심 주의 포인트
  • 📌 dst 생성 후 변경: dst[2] = 7 → key 2의 값이 4에서 7로 바뀜 → dst.values() = {2, 7, 6}
  • ➕ s.add(99) → s = {2, 4, 6, 99} (직접 99 추가)
  • ∩ 교집합: {2, 4, 6, 99} & {2, 7, 6} = {2, 6} — 원소 2개
  • 🖨️ len({2, 6}) = 2
