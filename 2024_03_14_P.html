Python type() 동적 타입 비교 & 분기 추적

Python · type() · str · float · int · tuple · isinstance · Tracing
Q문제 코드
Python 
def test(v):
    if   type(v) == type(""):    # str  인지 확인
        return len(v)
    elif type(v) == type(100):   # int  인지 확인  ← float과 다름!
        return 101
    else:                           # 그 외 모두 (float, tuple, list …)
        return 20

a = "100.0"        # str   — 따옴표 있음
b = 100.0          # float — 소수점 있음 (int 아님!)
c = (100.0, 200.0) # tuple
print(test(a) + test(b) + test(c))

위 코드의 출력값은 무엇인가?

print() 출력 결과
45
test("100.0")=5 + test(100.0)=20 + test((100.0,200.0))=20 = 45
① 문제 유형

Python의 동적 타입 시스템에서 type() 비교로 분기하는 함수를 세 가지 타입 인수로 추적하는 문제입니다.


  • ⚠️핵심 함정: b = 100.0은 float입니다. type(100.0) == type(100)은 float == int → False이므로 101이 아닌 20(else)을 반환합니다.
  • 🔷type("") = str, type(100) = int. float와 tuple은 두 조건 모두 불일치 → else 분기.
  • 🔷a = "100.0"은 문자열이므로 len("100.0") = 5를 반환합니다.

⑤ Python 숫자 타입 — float ≠ int 구분
⑥ 핵심 주의 포인트
  • ⚠️ 100과 100.0은 다른 타입: type(100) = int, type(100.0) = float. 소수점 하나로 완전히 다른 분기를 탑니다.
  • 📝 "100.0"은 숫자처럼 보이지만 따옴표로 감싸인 str입니다. len("100.0") = 5 (문자 5개).
  • 🔷 tuple, list, dict, float 등 str/int 외 모든 타입은 else 분기(→ 20)를 탑니다.
  • 🖨️ 최종 출력: 5 + 20 + 20 = 45
