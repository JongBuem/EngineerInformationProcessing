Algorithm Study Note

C 구조체 연결 리스트 포인터 탐색 추적

C · Struct · Linked List · Arrow Operator(->) · Pointer Chain · Tracing
Q문제 코드
C 
#include <stdio.h>

struct node {
    int          n1;   // 데이터 필드
    struct node *n2;   // 다음 노드 포인터
};

int main() {
    struct node a = {10, 0};   // a.n1=10, a.n2=NULL
    struct node b = {20, 0};   // b.n1=20, b.n2=NULL
    struct node c = {30, 0};   // c.n1=30, c.n2=NULL

    struct node *head = &a;    // head → a
    a.n2 = &b;                 // a.n2 → b
    b.n2 = &c;                 // b.n2 → c

    printf("%d", head->n2->n1);  // ← 이 값은?

    return 0;
}

위 코드의 출력값은 무엇인가?

printf() 출력 결과
20
head → a  |  head->n2 = a.n2 = &b  |  b.n1 = 20
① 문제 유형

자기 참조 구조체(Self-referential struct)로 구성한 단방향 연결 리스트에서, 화살표 연산자(->)를 두 번 연쇄 적용하여 특정 노드의 값을 읽는 문제입니다.


  • 🔷ptr->field는 (*ptr).field와 완전히 동일합니다. 포인터가 가리키는 구조체의 멤버에 한 번에 접근하는 축약 표기입니다.
  • ⚠️함정: ->가 두 번 이어져 있으므로 이동은 두 단계입니다. head→노드a(1hop)→노드b(2hop). 노드c(30)까지는 도달하지 않습니다.
② 연결 리스트 메모리 구조
③ 포인터 설정 단계 추적
구문 의미 결과
struct node a = {10, 0} a.n1=10, a.n2=NULL 노드 a 초기화
struct node b = {20, 0} b.n1=20, b.n2=NULL 노드 b 초기화
struct node c = {30, 0} c.n1=30, c.n2=NULL 노드 c 초기화
struct node *head = &a head에 a의 주소 저장 head → a
a.n2 = &b a.n2에 b의 주소 저장 a.n2 → b
b.n2 = &c b.n2에 c의 주소 저장 b.n2 → c
④ head->n2->n1 체인 단계별 분해
①
head
head 포인터 변수 → &a (노드 a의 메모리 주소 보유)
현재 위치: 노드 a
②
head->n2
= (*head).n2 = a.n2 = &b
노드 a의 n2 필드가 담고 있는 주소 = 노드 b의 주소
현재 위치: 노드 b
③
head->n2->n1
= (*(head->n2)).n1 = b.n1
노드 b의 n1 필드 값을 읽음
= 20
⑤ 화살표 연산자 등가 변환 비교
표현식 (*ptr).field 등가 변환 최종값
head->n2 (*head).n2 a.n2 = &b
head->n2->n1 (*(head->n2)).n1 b.n1 = 20
head->n2->n2->n1 한 칸 더 이동하는 경우 c.n1 = 30 (미해당)
⑥ 핵심 주의 포인트
  • ➡️ 화살표 연산자 ->는 (*ptr).field의 축약입니다. 포인터 없이 구조체 변수에 직접 접근할 때는 .을 사용합니다.
  • 🔗 체인 탐색: head → 노드a → (a.n2=&b) → 노드b → b.n1 = 20. ->가 두 번이므로 딱 한 노드를 건너뜁니다.
  • 📌 만약 head->n2->n2->n1이었다면 노드 b를 경유해 노드 c까지 이동하여 30이 출력됩니다.
  • 🖨️ 최종 출력: 20
