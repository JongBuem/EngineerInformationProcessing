C static 변수 — 호출 간 상태 유지 추적

C · static variable · local variable · scope · lifetime · Tracing
Q문제 코드
C 
#include <stdio.h>

int increment() {
    static int x = 0;   // 프로그램 시작 시 한 번만 초기화. 호출 간 유지!
    x += 2;
    return x;
}

int main() {
    int x = 1;           // main의 지역 변수 x (increment의 static x와 별개!)
    int sum = 0;
    for (int i = 0; i < 4; i++) {
        x++;              // main의 x 증가 (increment()와 무관)
        sum += increment();
    }
    printf("%d", sum);
}

위 코드의 출력값은 무엇인가?

printf("%d", sum) 출력 결과
20
increment() 반환값: 2 + 4 + 6 + 8 = 20
① 문제 유형

static 지역 변수의 생명 주기(lifetime)와 변수 이름 충돌(name shadowing)을 이해하는 추적 문제입니다.


  • ⚠️핵심 함정: main()의 지역 변수 x와 increment()의 static int x는 이름만 같고 완전히 별개의 변수입니다.
  • 🔷static int x = 0: 함수 내부지만 프로그램 시작 시 딱 한 번 초기화되고, 이후 호출에서도 값이 유지됩니다.
  • 🔷main()의 x++는 increment()의 static x에 아무 영향도 주지 않습니다.
② static 변수 vs 지역 변수 비교
STATIC STORAGE (데이터 영역)
increment()의 x0 → 2 → 4 → 6 → 8

• 함수 종료 후에도 값 유지
• 다음 호출 때 이전 값 사용
STACK (스택 영역)
main()의 x1 → 2 → 3 → 4 → 5
main()의 sum0 → 2 → 6 → 12 → 20
• 블록 진입 시 생성
• 블록 종료 시 소멸
• 매 호출마다 새로 생성

⑤ 최종 합산 수식

sum = 2 + 4 + 6 + 8 = 20
등차수열: 초항 2, 공차 2, 항수 4 → 4×(2+8)/2 = 4×5 = 20

⑥ 핵심 주의 포인트
  • 📌 static 변수는 딱 한 번 초기화: static int x = 0은 프로그램 시작 시 1회만 실행됩니다. 이후 호출에서는 이 줄을 무시하고 이전 값을 그대로 사용합니다.
  • ⚠️ 이름이 같아도 별개의 변수: main()의 x와 increment()의 static x는 서로 영향을 주지 않습니다. main의 x++는 sum에 영향이 없습니다.
  • 🔢 static x는 호출마다 +2씩 누적: 2, 4, 6, 8 → 합계 20
