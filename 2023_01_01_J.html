Java static 필드 vs 인스턴스 필드 — 후위 증가 출력 추적

Java · static field · instance field · post-increment · Tracing
Q문제 코드
Java 
class Static {
    public int a = 20;    // 인스턴스 필드
    static int b = 0;     // ★ 클래스(static) 필드 — 모든 인스턴스가 공유!
}

public class Test {
    public static void main(String[] args) {
        int a = 10;             // 지역 변수
        Static.b = a;           // Static.b = 10
        Static st = new Static(); // st.a=20, Static.b=10

        System.out.println(Static.b++); // ① 10 출력 후 b → 11
        System.out.println(st.b);       // ② st.b == Static.b == 11
        System.out.println(a);           // ③ 지역변수 a = 10
        System.out.print(st.a);         // ④ 인스턴스 필드 a = 20
    }
}

① 문제 유형

static 필드(클래스 공유)와 인스턴스 필드의 차이, 후위 증가 연산자(b++)의 출력 시점을 추적하는 문제입니다.

⑤ 핵심 포인트
  • 🔗 static 필드는 클래스에 하나만 존재하며 모든 인스턴스가 공유합니다. Static.b와 st.b는 같은 메모리입니다.
  • ⬆️ 후위 증가(b++)는 현재 값을 먼저 사용(출력)한 뒤 1 증가합니다. → println 시점에 10, 이후 b=11.
  • 📦 st.a(인스턴스 필드)와 지역변수 a는 이름이 같아도 서로 다른 변수입니다. st.a=20, 지역 a=10.
