Java 객체 참조 배열 스왑 & 필드 변경 추적

Java · Object Reference · Array · Swap · Heap · Tracing
Q문제 코드
Java 
public class Main {
    public static class BO {
        public int v;
        public BO(int v) { this.v = v; }
    }
    public static void main(String[] args) {
        BO a = new BO(1);
        BO b = new BO(2);
        BO c = new BO(3);
        BO[] arr = {a, b, c};    // arr[0]→a객체, arr[1]→b객체, arr[2]→c객체

        BO t  = arr[0];          // t → a객체
        arr[0] = arr[2];        // arr[0] → c객체
        arr[2] = t;             // arr[2] → a객체  (스왑 완료)
        arr[1].v = arr[0].v;   // b객체.v = c객체.v = 3

        System.out.println(a.v + " " + b.v + " " + c.v);
    }
}

위 코드의 출력값은 무엇인가?

System.out.println() 출력 결과
1 3 3
a.v=1 (불변) │ b.v=3 (arr[0].v=c.v=3으로 변경) │ c.v=3 (불변)
① 문제 유형

객체 참조 배열에서 참조 스왑과 실제 객체 필드 변경을 구별하여 추적하는 문제입니다.


  • 🔷참조 스왑 — arr[0]과 arr[2]가 가리키는 객체를 교환합니다. 원본 변수 a, c는 영향 없습니다.
  • 🔷필드 변경 — arr[1].v = arr[0].v는 b 객체의 v 필드를 직접 수정합니다.
  • 🔷출력은 arr이 아닌 a, b, c 변수로 합니다. 참조가 어느 객체를 가리키는지 정확히 추적해야 합니다.
② 메모리 구조 — 스택(참조변수) vs 힙(객체)
③ 단계별 변수 추적
단계 실행 구문 arr[0] arr[1] arr[2] a.v b.v c.v
초기 arr = {a, b, c} → obj_a → obj_b → obj_c 1 2 3
① t = arr[0] → obj_a → obj_b → obj_c 1 2 3
② arr[0] = arr[2] → obj_c → obj_b → obj_c 1 2 3
③ arr[2] = t → obj_c → obj_b → obj_a 1 2 3
④ arr[1].v = arr[0].v → obj_c → obj_b → obj_a 1 3 ← 변경! 3
출력 a.v + b.v + c.v — 1 3 3
④ 핵심 함정 — 참조 스왑 vs 필드 변경
연산 무엇이 바뀌나? 원본 변수 영향
arr[0] = arr[2] 배열 칸이 가리키는 참조값만 교체 없음 (a, c 불변)
arr[1].v = arr[0].v obj_b의 필드 v를 직접 수정 b.v 변경됨 → 3
⑤ 핵심 주의 포인트
  • 📌 arr는 객체를 담는 게 아니라 참조(주소)를 담습니다. 스왑해도 원본 객체 자체는 변하지 않습니다.
  • 🔄 스왑 후 arr = {obj_c, obj_b, obj_a} 순서로 재배치됩니다. 그러나 a, b, c 변수는 여전히 원래 객체를 가리킵니다.
  • ✏️ arr[1].v = arr[0].v — arr[1]은 obj_b(= b가 가리키는 객체)이므로 b.v가 3으로 변경됩니다.
  • 🖨️ 출력: a.v=1 (obj_a 불변), b.v=3 (필드 변경됨), c.v=3 (obj_c 불변) → "1 3 3"
