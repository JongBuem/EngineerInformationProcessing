3문자열과 포인터 산술
C 
char* s = "BEST";
// 메모리: 'B' 'E' 'S' 'T' '\0'
연산 결과 설명
*s 'B' s가 가리키는 첫 문자
*(s+1) 'E' 한 칸 앞 문자
s[2] 'S' = *(s+2), 첨자 표기

→ 문자열 순회의 기본 패턴. '\0'(null)이 나오면 false가 되어 반복 종료.

4연결 리스트 핵심 패턴
① 기본 구조
struct node {
  int data;
  struct node* next;
};
② 순회 (traversal)
node* cur = head;
while (cur != NULL) {
  printf("%d ", cur->data);
  cur = cur->next;
}
③ 노드 삭제/재연결
// target 노드를 건너뜀
prev->next
  = target->next;
④ 헤드 삽입
new_node->next = head;
head = new_node;
→ 출력: 삽입 역순!
⑤ 인접 값 교환
int tmp = a->data;
a->data = b->data;
b->data = tmp;
포인터(next)는 그대로!
  • ⚠️ 헤드 삽입(head insert)은 새 노드를 맨 앞에 추가합니다. 1→2→3 순으로 삽입하면 출력은 3→2→1 (역순).
  • ⚠️ 값만 교환 vs 포인터 교환: 값만 바꾸면 연결 순서(next 포인터)는 그대로이므로 출력 순서는 연결 순서 기준입니다.

8재귀 함수(Recursion) 핵심 패턴

재귀 함수는 자기 자신을 호출하는 함수입니다. 시험에서는 실행 흐름과 반환값 추적이 핵심이며, 반드시 기저 조건(base case)이 존재해야 무한 루프를 막을 수 있습니다.

1재귀의 2가지 필수 요소
요소 설명 예시 (factorial)
기저 조건 (Base Case) 재귀를 멈추는 조건 — 없으면 무한 호출 if (n <= 1) return 1;
재귀 호출 (Recursive Case) 문제를 더 작게 만들어 자신을 호출 return n * f(n-1);
2팩토리얼 — 기본 재귀 패턴
C 
int f(int n) {
    if (n <= 1) return 1;     // 기저 조건
    return n * f(n - 1);        // 재귀 호출
}
// f(4) = 4 * f(3)
//      = 4 * 3 * f(2)
//      = 4 * 3 * 2 * f(1)
//      = 4 * 3 * 2 * 1  = 24
3시험 자주 출제 — 재귀 출력 순서 패턴
C — 출력 위치에 따른 차이 
// ① 재귀 호출 전 출력 → 호출 순서대로 출력
void pre(int n) {
    if (n == 0) return;
    printf("%d ", n);   // 먼저 출력
    pre(n - 1);          // 재귀 후
}
// pre(3) → 출력: 3 2 1

// ② 재귀 호출 후 출력 → 역순 출력
void post(int n) {
    if (n == 0) return;
    post(n - 1);          // 재귀 먼저
    printf("%d ", n);    // 나중에 출력
}
// post(3) → 출력: 1 2 3
4재귀 함수 추적 요령
  • 📐호출 트리를 직접 그린다. 각 노드에 인자값과 반환값을 기록한다.
  • 🔴기저 조건부터 올라오며 값을 계산한다. 가장 깊은 호출이 먼저 반환된다.
  • 📍printf 위치가 재귀 호출 앞/뒤 중 어디인지 확인 — 출력 순서가 완전히 달라진다.
  • ⚠️기저 조건 부등호를 정확히 확인: <=1과 ==0은 같은 결과일 수도, 다를 수도 있다.
